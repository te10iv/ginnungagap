# Aurora 概要

## 1. サービスの役割

Amazon Auroraは、クラウド向けに構築された高性能なリレーショナルデータベースエンジンです。MySQLやPostgreSQLと互換性があり、RDSの5倍のパフォーマンス、自動スケーリング、高可用性を提供します。

## 2. 主なユースケース

- 高性能が求められるアプリケーション
- 大規模なデータベースワークロード
- 高可用性が重要なアプリケーション
- 自動スケーリングが必要なアプリケーション
- グローバルデータベース（複数リージョンでのレプリケーション）
- サーバーレスモード（オンデマンドでスケーリング）

## 3. 他サービスとの関係

- **RDS**: RDSのデータベースエンジンの一つ
- **EC2**: アプリケーションサーバーからAuroraに接続
- **VPC**: AuroraクラスターをVPC内に配置
- **Security Group**: データベースへのアクセス制御
- **CloudWatch**: メトリクスとログの監視
- **Lambda**: データベースの変更イベントでトリガー
- **S3**: データのエクスポート・インポート

## 4. 料金のざっくりイメージ

- **インスタンス**: インスタンスタイプにより異なる（例: db.t3.mediumで約$0.082/時間 = 約$59/月）
- **ストレージ**: GBあたり$0.10/月（自動スケーリング、最大128TB）
- **IOPS**: 無料（ストレージに含まれる）
- **サーバーレス**: ACU（Aurora Capacity Unit）あたり$0.06/時間
- **注意**: RDSより高価だが、パフォーマンスとスケーラビリティが優れている

## 5. 初心者がまず覚えるべきポイント

- MySQL/PostgreSQL互換の高性能データベース
- ストレージが自動スケーリング（最大128TB）
- 最大15個のリードレプリカ（RDSは最大5個）
- マルチマスター構成が可能（Aurora Multi-Master）
- グローバルデータベース（複数リージョンでのレプリケーション）
- サーバーレスモード（使用しない時は課金されない）
- バックトラッキング（過去の任意の時点に戻れる）
- 高速クローン（数分でデータベースを複製）
- RDSより高価だが、パフォーマンスと機能が優れている

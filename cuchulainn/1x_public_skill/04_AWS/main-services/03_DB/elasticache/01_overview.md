# ElastiCache 概要

## 1. サービスの役割

Amazon ElastiCacheは、メモリ内データストアとキャッシュを提供するマネージドサービスです。RedisとMemcachedをサポートし、アプリケーションのパフォーマンス向上とデータベースの負荷軽減を実現します。

## 2. 主なユースケース

- データベースのクエリ結果をキャッシュ（読み取り負荷の軽減）
- セッションストア（ユーザーセッション情報の保存）
- リアルタイムアプリケーション（ゲーム、チャットなど）
- レート制限（APIの呼び出し回数制限）
- パブリッシュ/サブスクライブ（Pub/Sub）パターン
- ランキングやリーダーボード

## 3. 他サービスとの関係

- **RDS, Aurora**: データベースのクエリ結果をキャッシュ
- **EC2, Lambda**: アプリケーションからElastiCacheに接続
- **VPC**: ElastiCacheクラスターをVPC内に配置
- **Security Group**: キャッシュへのアクセス制御
- **CloudWatch**: メトリクスとログの監視
- **Auto Scaling**: ノード数の自動スケーリング

## 4. 料金のざっくりイメージ

- **インスタンス**: インスタンスタイプにより異なる（例: cache.t3.microで約$0.017/時間 = 約$12/月）
- **データ転送**: リージョン外への転送で料金発生
- **バックアップ**: ストレージ料金（S3に保存）
- **注意**: メモリベースなので、インスタンスサイズが大きいほど高価

## 5. 初心者がまず覚えるべきポイント

- RedisとMemcachedの2つのエンジンがある
- **Redis**: より多くの機能（Pub/Sub、永続化、レプリケーションなど）
- **Memcached**: シンプルで高速、マルチスレッド対応
- クラスターモード（Redis）で水平スケーリング
- レプリケーションで高可用性を実現
- マルチAZ配置で可用性を向上
- 自動フェイルオーバー（Redis）
- バックアップとリストアが可能（Redis）
- セキュリティグループでアクセス制御
- データベースの負荷を軽減し、アプリケーションのパフォーマンスを向上

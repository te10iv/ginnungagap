# ElastiCache マネージドコンソールでのセットアップ

## 作成するもの

ElastiCache Redisクラスターを作成し、アプリケーションのキャッシュ層として使用します。データベースのクエリ結果をキャッシュしてパフォーマンスを向上させます。

## セットアップ手順

1. **ElastiCacheコンソールを開く**
   - AWSマネージメントコンソールで「ElastiCache」を検索して選択

2. **クラスターを作成**
   - 「Redisクラスターを作成」をクリック

3. **クラスター設定**
   - **名前**: `my-redis-cluster` を入力
   - **説明**: 任意の説明を入力

4. **ノードタイプの設定**
   - **ノードタイプ**: cache.t3.microを選択（テスト用）

5. **ネットワーク設定**
   - **VPC**: 作成済みのVPCを選択
   - **サブネットグループ**: デフォルトを選択
   - **セキュリティグループ**: 新しいセキュリティグループを作成（Redisポート6379を許可）

6. **その他の設定**
   - **ポート**: 6379（デフォルト）
   - **パラメータグループ**: デフォルトを選択

7. **確認と作成**
   - 設定を確認
   - 「クラスターを作成」をクリック

8. **確認**
   - クラスターの状態が「利用可能」になるまで待機
   - エンドポイントをメモ

## 補足

- Redisクラスターはプライベートサブネットに配置することを推奨します
- エンドポイントを使用してアプリケーションから接続します

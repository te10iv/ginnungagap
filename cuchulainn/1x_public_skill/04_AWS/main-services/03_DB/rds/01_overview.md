# RDS 概要

## 1. サービスの役割

Amazon RDS（Relational Database Service）は、クラウドでリレーショナルデータベースを簡単にセットアップ、運用、スケールできるマネージドサービスです。MySQL、PostgreSQL、MariaDB、Oracle、SQL Server、Auroraなどのデータベースエンジンをサポートします。

## 2. 主なユースケース

- Webアプリケーションのデータベース
- エンタープライズアプリケーションのバックエンド
- データウェアハウス（小規模）
- 開発・テスト環境のデータベース
- レプリケーションによる高可用性構成
- 自動バックアップとポイントインタイムリカバリ

## 3. 他サービスとの関係

- **EC2**: アプリケーションサーバーからRDSに接続
- **VPC**: RDSインスタンスをVPC内に配置
- **Security Group**: データベースへのアクセス制御
- **CloudWatch**: メトリクスとログの監視
- **S3**: スナップショットの保存
- **Lambda**: データベースの変更イベントでトリガー
- **ElastiCache**: キャッシュ層として使用

## 4. 料金のざっくりイメージ

- **インスタンス**: インスタンスタイプにより異なる（例: db.t3.microで約$0.017/時間 = 約$12/月）
- **ストレージ**: GBあたり$0.115/月（gp3の場合）
- **IOPS**: プロビジョンドIOPSの場合、追加料金
- **データ転送**: リージョン外への転送で料金発生
- **注意**: エンジンによって料金が異なる（Auroraは別料金体系）

## 5. 初心者がまず覚えるべきポイント

- マネージドサービス（OS、パッチ適用、バックアップは自動）
- 複数のデータベースエンジンに対応（MySQL、PostgreSQL、MariaDB、Oracle、SQL Server、Aurora）
- マルチAZ配置で高可用性を実現
- 自動バックアップ（デフォルトで7日間保持）
- ポイントインタイムリカバリ（PITR）が可能
- リードレプリカで読み取り負荷を分散
- セキュリティグループでアクセス制御
- パラメータグループでデータベース設定を管理
- スナップショットで手動バックアップも可能

# DynamoDB 概要

## 1. サービスの役割

Amazon DynamoDBは、完全マネージドのNoSQLデータベースサービスです。サーバーレスで、自動スケーリング、高可用性、低レイテンシーを提供します。キー・バリュー型とドキュメント型のデータモデルをサポートします。

## 2. 主なユースケース

- サーバーレスアプリケーションのデータストア
- リアルタイムアプリケーション（ゲーム、IoTなど）
- 高スループットが必要なアプリケーション
- スケーラビリティが重要なアプリケーション
- モバイルアプリケーションのバックエンド
- セッションストアやキャッシュ

## 3. 他サービスとの関係

- **Lambda**: DynamoDBの変更イベントでトリガー
- **API Gateway**: REST APIのバックエンドとして使用
- **AppSync**: GraphQL APIのデータソースとして使用
- **CloudWatch**: メトリクスとログの監視
- **IAM**: テーブルへのアクセス制御
- **S3**: データのエクスポート・インポート
- **Streams**: テーブルの変更をストリーミング

## 4. 料金のざっくりイメージ

- **オンデマンド**: リクエスト数に応じて自動課金（書き込み: $1.25/100万リクエスト、読み取り: $0.25/100万リクエスト）
- **プロビジョンド**: キャパシティユニットを事前に設定（書き込み: $1.25/100万WCU/月、読み取り: $0.25/100万RCU/月）
- **ストレージ**: GBあたり$0.25/月
- **バックアップ**: GBあたり$0.20/月
- **無料利用枠**: 毎月25GBストレージ、200万読み取り、100万書き込み

## 5. 初心者がまず覚えるべきポイント

- 完全マネージドのNoSQLデータベース（サーバー管理不要）
- 自動スケーリング（オンデマンドモード）
- 単一桁ミリ秒のレイテンシー
- パーティションキーとソートキーでデータを管理
- グローバルテーブル（複数リージョンでのレプリケーション）
- TTL（Time To Live）でデータを自動削除
- DynamoDB Streamsで変更イベントを処理
- ポイントインタイムリカバリ（PITR）が可能
- オンデマンドモードとプロビジョンドモードの使い分け

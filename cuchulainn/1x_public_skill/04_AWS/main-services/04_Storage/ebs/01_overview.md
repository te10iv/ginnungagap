# EBS 概要

## 1. サービスの役割

Amazon EBS（Elastic Block Store）は、EC2インスタンス用の永続的なブロックストレージボリュームです。EC2インスタンスにアタッチして使用し、データの永続化、スナップショット、暗号化などの機能を提供します。

## 2. 主なユースケース

- EC2インスタンスのルートボリューム（OSディスク）
- データベースのデータストレージ
- アプリケーションのデータ保存
- ログファイルの保存
- スナップショットによるバックアップ
- ボリュームの拡張とパフォーマンス調整

## 3. 他サービスとの関係

- **EC2**: EBSボリュームをEC2インスタンスにアタッチ
- **S3**: スナップショットをS3に保存
- **Snapshot**: EBSボリュームのバックアップ
- **VPC**: EBSボリュームはVPC内のEC2インスタンスにアタッチ
- **CloudWatch**: ボリュームのメトリクス監視
- **KMS**: ボリュームの暗号化

## 4. 料金のざっくりイメージ

- **gp3（General Purpose SSD）**: GBあたり$0.08/月、3,000 IOPS/125 MB/sまで無料
- **gp2（General Purpose SSD）**: GBあたり$0.10/月（IOPSは容量に比例）
- **io1/io2（Provisioned IOPS SSD）**: GBあたり$0.125/月 + IOPSあたり$0.10/月
- **st1（Throughput Optimized HDD）**: GBあたり$0.045/月
- **sc1（Cold HDD）**: GBあたり$0.015/月
- **スナップショット**: GBあたり$0.05/月

## 5. 初心者がまず覚えるべきポイント

- EC2インスタンス用のブロックストレージ
- ボリュームタイプ（gp3、gp2、io1、io2、st1、sc1）で用途に応じて選択
- **gp3**: 汎用、コストパフォーマンスが良い（推奨）
- **io1/io2**: 高IOPSが必要なデータベースなど
- **st1/sc1**: 大容量、低コスト（バックアップなど）
- スナップショットでバックアップ（増分バックアップ）
- 暗号化（デフォルトで有効化可能）
- ボリュームの拡張（サイズとIOPS）
- マルチアタッチ（io1/io2のみ、複数インスタンスから同時アクセス）
- インスタンス終了時に削除するかどうかを設定可能

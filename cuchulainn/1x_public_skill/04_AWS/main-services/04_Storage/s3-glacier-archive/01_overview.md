# S3 Glacier / Archive 概要

## 1. サービスの役割

Amazon S3 GlacierとS3 Glacier Deep Archiveは、長期アーカイブストレージサービスです。低コストでデータを長期保存でき、コンプライアンス要件やバックアップ用途に適しています。データの取得には時間がかかりますが、保存コストは非常に安価です。

## 2. 主なユースケース

- 長期バックアップ（データベース、ファイルシステムなど）
- コンプライアンス要件を満たすためのデータアーカイブ
- 災害復旧用のデータ保存
- メディアアセットのアーカイブ
- ログファイルの長期保存
- レガシーデータの保存

## 3. 他サービスとの関係

- **S3**: S3のストレージクラスとして使用（ライフサイクルポリシーで自動移行）
- **Lambda**: Glacierからのデータ取得をトリガー
- **CloudWatch**: アーカイブと取得のメトリクス監視
- **IAM**: アーカイブへのアクセス制御
- **S3 Inventory**: アーカイブされたオブジェクトの一覧

## 4. 料金のざっくりイメージ

- **S3 Glacier Flexible Retrieval**: GBあたり$0.0036/月（取得: Expedited $0.03/GB、Standard $0.01/GB、Bulk $0.0025/GB）
- **S3 Glacier Deep Archive**: GBあたり$0.00099/月（取得: Standard 12時間 $0.02/GB、Bulk 48時間 $0.0025/GB）
- **注意**: 取得時間が長いほど安価（Expedited < Standard < Bulk）

## 5. 初心者がまず覚えるべきポイント

- 長期アーカイブストレージ（低コスト、取得に時間がかかる）
- **S3 Glacier Flexible Retrieval**: 数分〜数時間で取得可能、月1回の無料取得
- **S3 Glacier Deep Archive**: 最も安価、12時間〜48時間で取得
- ライフサイクルポリシーでS3 Standardから自動移行
- バルク取得が最も安価（大量データの取得に適している）
- 最小保存期間（90日または180日）がある
- 早期削除料金（最小保存期間前に削除すると追加料金）
- バージョニングと暗号化に対応
- コンプライアンス要件を満たすための長期保存に適している

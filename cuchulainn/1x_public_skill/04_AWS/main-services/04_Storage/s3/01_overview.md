# S3 概要

## 1. サービスの役割

Amazon S3（Simple Storage Service）は、オブジェクトストレージサービスです。任意の量のデータを保存・取得でき、99.999999999%（11個の9）の耐久性と高い可用性を提供します。静的ウェブサイトホスティング、データレイク、バックアップなど様々な用途に使用されます。

## 2. 主なユースケース

- 静的ウェブサイトホスティング（HTML、CSS、JavaScript、画像など）
- データレイク（ビッグデータ分析のためのデータ保存）
- バックアップとアーカイブ
- コンテンツ配信（CloudFrontと連携）
- データのエクスポート・インポート
- ログファイルの保存
- メディアファイル（画像、動画、音声）の保存

## 3. 他サービスとの関係

- **CloudFront**: コンテンツ配信ネットワークと連携
- **Lambda**: S3イベント（アップロード、削除など）でトリガー
- **Glacier**: アーカイブストレージとして使用
- **Athena**: S3のデータを直接クエリ
- **EC2, ECS**: インスタンスからS3にアクセス
- **IAM**: バケットとオブジェクトへのアクセス制御
- **EventBridge**: S3イベントの通知

## 4. 料金のざっくりイメージ

- **ストレージ**: GBあたり$0.023/月（Standard、最初の50TB）
- **リクエスト**: PUT/COPY/POST: $0.005/1,000リクエスト、GET: $0.0004/1,000リクエスト
- **データ転送**: リージョン外への転送で料金発生
- **無料利用枠**: 最初の5GBストレージ、20,000 GETリクエスト、2,000 PUTリクエスト

## 5. 初心者がまず覚えるべきポイント

- オブジェクトストレージ（ファイルを保存するサービス）
- バケット（コンテナ）にオブジェクト（ファイル）を保存
- リージョンごとにバケットを作成
- ストレージクラス（Standard、IA、Glacierなど）でコスト最適化
- ライフサイクルポリシーで自動的にストレージクラスを移行
- バージョニングでオブジェクトの履歴を管理
- 暗号化（SSE-S3、SSE-KMS、SSE-C）でデータを保護
- アクセス制御（バケットポリシー、ACL、IAMポリシー）
- 静的ウェブサイトホスティングが可能
- イベント通知（SNS、SQS、Lambda、EventBridge）

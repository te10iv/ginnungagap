# Cognito 概要

## 1. サービスの役割

Amazon Cognitoは、モバイルアプリケーションやWebアプリケーションにユーザー認証、認可、ユーザー管理機能を追加するサービスです。ユーザープール（認証）とIDプール（認可）の2つの主要コンポーネントがあります。

## 2. 主なユースケース

- モバイルアプリケーションのユーザー認証
- Webアプリケーションのユーザー管理
- ソーシャルログイン（Google、Facebook、Amazonなど）
- 多要素認証（MFA）
- ユーザー属性の管理
- 一時的な認証情報の提供
- サーバーレスアプリケーションの認証

## 3. 他サービスとの関係

- **API Gateway**: 認証されたユーザーのみAPIにアクセス
- **Lambda**: 認証イベントの処理
- **S3, DynamoDB**: 認証されたユーザーのみリソースにアクセス
- **IAM**: 一時的な認証情報の提供
- **CloudFront**: 署名付きURLの生成

## 4. 料金のざっくりイメージ

- **ユーザープール**: MAU（Monthly Active Users）あたり$0.0055（最初の50,000 MAUは無料）
- **IDプール**: 認証あたり$0.0055（最初の50,000認証/月は無料）
- **SMS MFA**: SMSあたり$0.00645（米国内）
- **注意**: ユーザー数に応じて課金

## 5. 初心者がまず覚えるべきポイント

- ユーザー認証・認可サービス
- **ユーザープール**: ユーザーの認証（サインアップ、サインイン、パスワードリセットなど）
- **IDプール**: 認証されたユーザーに一時的なAWS認証情報を提供
- ソーシャルログイン（Google、Facebook、Amazon、Appleなど）
- 多要素認証（MFA）のサポート
- ユーザー属性のカスタマイズ
- カスタム認証フロー（Lambdaトリガー）
- ユーザー移行（既存のユーザーデータベースから移行）
- JWTトークンの発行
- サーバーレスアプリケーションの認証に適している

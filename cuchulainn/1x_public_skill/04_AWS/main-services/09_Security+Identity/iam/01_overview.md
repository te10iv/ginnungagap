# IAM 概要

## 1. サービスの役割

AWS Identity and Access Management（IAM）は、AWSリソースへのアクセスを安全に制御するサービスです。ユーザー、グループ、ロールを作成し、ポリシーで権限を管理します。認証（誰がアクセスできるか）と認可（何ができるか）を制御します。

## 2. 主なユースケース

- ユーザーとグループの管理
- ロールベースのアクセス制御（RBAC）
- AWSリソースへのアクセス権限の管理
- 最小権限の原則の実現
- 一時的な認証情報の提供（STS）
- マルチファクター認証（MFA）
- アクセスキーとシークレットキーの管理
- コンプライアンスとセキュリティ監査

## 3. 他サービスとの関係

- **すべてのAWSサービス**: IAMでアクセス制御
- **S3**: バケットとオブジェクトへのアクセス制御
- **EC2**: インスタンスプロファイル（IAMロール）
- **Lambda**: 実行ロール（IAMロール）
- **CloudTrail**: IAMアクティビティの監査
- **Organizations**: 組織全体のIAMポリシー管理

## 4. 料金のざっくりイメージ

- IAM自体は無料
- 関連するサービス（CloudTrail、MFAデバイスなど）に料金が発生する場合あり

## 5. 初心者がまず覚えるべきポイント

- AWSの認証・認可サービス（すべてのAWSサービスの基盤）
- **ユーザー**: 個人やアプリケーションの認証情報
- **グループ**: 複数のユーザーをまとめて管理
- **ロール**: 一時的な認証情報を提供（EC2、Lambdaなどで使用）
- **ポリシー**: 権限を定義するJSONドキュメント
- 最小権限の原則（必要最小限の権限のみ付与）
- ルートアカウントは使用しない（IAMユーザーを作成）
- アクセスキーは定期的にローテーション
- MFA（多要素認証）を有効化
- ポリシーの評価ロジック（明示的な拒否 > 明示的な許可 > 暗黙的な拒否）

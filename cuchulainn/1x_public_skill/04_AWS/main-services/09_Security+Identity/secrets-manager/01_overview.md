# Secrets Manager 概要

## 1. サービスの役割

AWS Secrets Managerは、データベースの認証情報、APIキー、その他の機密情報を安全に保存・管理するサービスです。シークレットの自動ローテーション、アクセス制御、監査ログなどの機能を提供します。

## 2. 主なユースケース

- データベースの認証情報（ユーザー名、パスワード）の管理
- APIキーやトークンの保存
- アプリケーションの設定情報の管理
- シークレットの自動ローテーション
- コンプライアンス要件を満たすための機密情報管理
- マルチリージョンでのシークレットのレプリケーション

## 3. 他サービスとの関係

- **RDS, Aurora**: データベースの認証情報を自動ローテーション
- **Lambda**: シークレットを取得して使用
- **ECS, EKS**: コンテナからシークレットを取得
- **KMS**: シークレットの暗号化
- **IAM**: シークレットへのアクセス制御
- **CloudTrail**: シークレットのアクセスを監査

## 4. 料金のざっくりイメージ

- **シークレット**: シークレットあたり$0.40/月
- **APIリクエスト**: 最初の10,000リクエスト/月は無料、以降$0.05/10,000リクエスト
- **注意**: Parameter Storeより高価だが、自動ローテーション機能がある

## 5. 初心者がまず覚えるべきポイント

- 機密情報（シークレット）の管理サービス
- Parameter Storeより高価だが、自動ローテーション機能がある
- データベースの認証情報を自動ローテーション（RDS、Aurora、Redshift、DocumentDB）
- シークレットのバージョン管理
- IAMでシークレットへのアクセスを制御
- KMSでシークレットを暗号化
- Lambda関数からシークレットを取得可能
- CloudTrailでシークレットのアクセスを監査
- マルチリージョンでのレプリケーション
- JSON形式でシークレットを保存可能

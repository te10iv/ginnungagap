# Config マネージドコンソールでのセットアップ

## 作成するもの

AWS Configを有効化し、AWSリソースの設定変更を記録・監査します。コンプライアンスルールを評価します。

## セットアップ手順

1. **Configコンソールを開く**
   - AWSマネージメントコンソールで「Config」を検索して選択

2. **設定を開始**
   - 「今すぐ始める」をクリック

3. **設定の記録を設定**
   - **リソースタイプ**: すべてのリソースを記録を選択
   - **記録の頻度**: 継続的を選択

4. **ストレージの場所を設定**
   - **S3バケット**: 新しいバケットを作成または既存のバケットを選択
   - **S3キープレフィックス**: `config/` を入力
   - **SNSトピック**: 新しいトピックを作成または既存のトピックを選択

5. **ルールを設定**
   - **ルール**: 推奨されるルールを選択（例: root-access-key-check）
   - または後でルールを追加

6. **確認と設定**
   - 設定を確認
   - 「確認」をクリック

7. **コンプライアンスダッシュボードを確認**
   - ダッシュボードでリソースのコンプライアンス状態を確認

## 補足

- Configはリソースの設定変更を自動的に記録します
- カスタムルールをLambda関数で作成可能です

# Config 概要

## 1. サービスの役割

AWS Configは、AWSリソースの設定を記録、評価、監査するサービスです。リソースの設定変更を追跡し、コンプライアンスルールに基づいて評価を行い、コンプライアンス違反を検出します。

## 2. 主なユースケース

- AWSリソースの設定変更の追跡
- コンプライアンスルールの評価
- セキュリティポリシーの監視
- リソースの設定履歴の管理
- コンプライアンスレポートの生成
- リソースの設定変更の通知

## 3. 他サービスとの関係

- **すべてのAWSサービス**: リソースの設定を記録
- **CloudTrail**: API呼び出しと連携
- **S3**: 設定スナップショットと履歴の保存
- **SNS**: コンプライアンス違反の通知
- **Lambda**: カスタムルールの評価
- **Organizations**: 組織単位でのコンプライアンス管理

## 4. 料金のざっくりイメージ

- **設定アイテム**: 設定アイテムあたり$0.003（最初の100,000設定アイテム/月は無料）
- **設定スナップショット**: スナップショットあたり$0.003
- **カスタムルール**: ルール評価あたり$0.001（最初の100,000評価/月は無料）
- **注意**: リソース数とルール評価数に応じて課金

## 5. 初心者がまず覚えるべきポイント

- AWSリソースの設定管理とコンプライアンスサービス
- **設定アイテム**: リソースの設定スナップショット
- **設定履歴**: リソースの設定変更の履歴
- **マネージドルール**: AWSが提供するコンプライアンスルール
- **カスタムルール**: Lambda関数でカスタムルールを作成
- コンプライアンスダッシュボードで環境の状態を確認
- 設定変更の通知（SNS経由）
- リソースの設定変更の可視化
- コンプライアンスレポートの生成
- 組織単位でのコンプライアンス管理（Organizationsと連携）

# AWS Config：まずこれだけ（Lv1）

## このサービスの一言説明
- AWS Config は「**リソース設定履歴とコンプライアンス監査**」を提供するAWSサービス

## ゴール（ここまでできたら合格）
- Config を **有効化できる**
- **設定変更履歴を確認できる**
- 「コンプライアンス監査にはConfigが必要」と判断できる

## まず覚えること（最小セット）
- **設定変更履歴**：リソース設定変更記録
- **Config Rules**：コンプライアンスルール
- **Conformance Pack**：ルール集
- **リソース関係**：リソース間依存関係表示
- **S3保存**：設定履歴S3保存

## できるようになること
- □ Config有効化できる
- □ 設定変更履歴確認できる
- □ Config Rules設定できる
- □ コンプライアンス違反確認できる

## まずやること（Hands-on）
- Config有効化
- マネージドルール追加（S3暗号化必須等）
- リソース作成
- コンプライアンスダッシュボード確認

## 関連するAWSサービス（名前だけ）
- **S3**：設定履歴保存
- **SNS**：変更通知
- **Lambda**：カスタムルール
- **Organizations**：組織ルール
- **Systems Manager**：自動修復

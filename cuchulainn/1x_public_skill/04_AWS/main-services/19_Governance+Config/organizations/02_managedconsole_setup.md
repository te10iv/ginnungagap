# Organizations マネージドコンソールでのセットアップ

## 作成するもの

AWS Organizationsを有効化し、複数のAWSアカウントを一元管理します。組織単位での一括請求とポリシー管理を設定します。

## セットアップ手順

1. **Organizationsコンソールを開く**
   - AWSマネージメントコンソールで「Organizations」を検索して選択

2. **組織を作成**
   - 「組織を作成」をクリック
   - 組織の作成を確認

3. **組織単位（OU）を作成**
   - 「組織単位」を選択
   - 「組織単位を作成」をクリック
   - **名前**: `Production` を入力
   - **親**: ルートを選択
   - 「組織単位を作成」をクリック

4. **アカウントを招待（既存アカウントの場合）**
   - 「アカウント」を選択
   - 「アカウントを招待」をクリック
   - **メールアドレス**: 招待するアカウントのメールアドレスを入力
   - 「招待を送信」をクリック

5. **アカウントを作成（新規アカウントの場合）**
   - 「アカウント」を選択
   - 「アカウントを追加」をクリック
   - **アカウント名**: `dev-account` を入力
   - **メールアドレス**: アカウントのメールアドレスを入力
   - **IAMロール名**: OrganizationAccountAccessRole（デフォルト）
   - 「アカウントを作成」をクリック

6. **サービス制御ポリシー（SCP）を作成（オプション）**
   - 「ポリシー」を選択
   - 「サービス制御ポリシーを作成」をクリック
   - **名前**: `deny-region-policy` を入力
   - **ポリシー**: JSONでポリシーを記述
   - 「ポリシーを作成」をクリック

## 補足

- マスターアカウントが組織を管理します
- 一括請求で複数アカウントの請求を統合できます

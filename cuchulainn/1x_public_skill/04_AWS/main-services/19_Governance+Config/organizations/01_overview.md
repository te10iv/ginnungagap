# Organizations 概要

## 1. サービスの役割

AWS Organizationsは、複数のAWSアカウントを一元管理するサービスです。組織単位でアカウントをグループ化し、一括請求、ポリシー管理、アクセス制御などを行います。

## 2. 主なユースケース

- 複数のAWSアカウントの一元管理
- 組織単位での一括請求
- セントラルポリシーの適用（SCP）
- アカウントの階層構造の構築
- コスト配分タグの管理
- サービス制御ポリシー（SCP）によるアクセス制御
- 組織単位でのリソース共有

## 3. 他サービスとの関係

- **すべてのAWSサービス**: 組織単位でポリシーを適用
- **IAM**: サービス制御ポリシー（SCP）でアクセス制御
- **CloudTrail**: 組織単位での監査
- **Config**: 組織単位でのコンプライアンス管理
- **Control Tower**: 組織のガバナンスとコンプライアンス

## 4. 料金のざっくりイメージ

- Organizations自体は無料
- 関連するサービス（CloudTrail、Configなど）に料金が発生する場合あり

## 5. 初心者がまず覚えるべきポイント

- 複数AWSアカウントの一元管理サービス
- **マスターアカウント**: 組織のルートアカウント
- **メンバーアカウント**: 組織に参加するアカウント
- **組織単位（OU）**: アカウントをグループ化
- **サービス制御ポリシー（SCP）**: アカウントやOUにポリシーを適用
- 一括請求（すべてのアカウントの請求をマスターアカウントに集約）
- 統合請求（複数アカウントの使用量を統合）
- リソース共有（RAM: Resource Access Manager）
- 組織単位でのCloudTrailとConfigの統合
- アカウントの自動作成と管理

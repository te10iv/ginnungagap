# AWS Organizations：まずこれだけ（Lv1）

## このサービスの一言説明
- AWS Organizations は「**複数AWSアカウントの一元管理**」を提供するAWSサービス

## ゴール（ここまでできたら合格）
- 組織を **作成できる**
- **アカウント追加できる**
- 「複数アカウント管理にはOrganizationsが必要」と判断できる

## まず覚えること（最小セット）
- **組織（Organization）**：複数アカウントの集合
- **組織単位（OU）**：アカウントのグループ
- **管理アカウント**：組織の親アカウント
- **メンバーアカウント**：組織配下のアカウント
- **一括請求**：全アカウント統合請求

## できるようになること
- □ 組織作成できる
- □ 新規アカウント作成できる
- □ OUを作成できる
- □ SCPを理解できる

## まずやること（Hands-on）
- Organizations有効化
- OU作成（開発、本番等）
- 新規アカウント作成

## 関連するAWSサービス（名前だけ）
- **IAM**：権限管理
- **CloudTrail**：組織全体ログ
- **Config**：組織全体設定
- **Control Tower**：ガバナンス自動化
- **SSO**：シングルサインオン

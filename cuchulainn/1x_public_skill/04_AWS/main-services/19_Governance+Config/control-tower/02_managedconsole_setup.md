# Control Tower マネージドコンソールでのセットアップ

## 作成するもの

AWS Control Towerをセットアップし、マルチアカウント環境のガバナンスとコンプライアンスを自動化します。ベストプラクティスに基づいた環境を構築します。

## セットアップ手順

1. **Control Towerコンソールを開く**
   - AWSマネージメントコンソールで「Control Tower」を検索して選択

2. **ランディングゾーンをセットアップ**
   - 「ランディングゾーンをセットアップ」をクリック

3. **ホームリージョンを選択**
   - **ホームリージョン**: ap-northeast-1を選択

4. **組織の設定**
   - **組織の管理**: 既存の組織を使用または新しい組織を作成

5. **ログアーカイブと監査アカウントの設定**
   - **ログアーカイブアカウント**: 新しいアカウントを作成
   - **監査アカウント**: 新しいアカウントを作成

6. **ガードレールの設定**
   - 推奨されるガードレールが自動的に有効化されます
   - 必要に応じて追加のガードレールを選択

7. **確認とセットアップ**
   - 設定を確認
   - 「ランディングゾーンをセットアップ」をクリック
   - セットアップには30分〜1時間かかります

8. **確認**
   - ダッシュボードで環境の状態を確認

## 補足

- Control TowerはOrganizationsと統合されています
- ガードレールでコンプライアンスを強制します

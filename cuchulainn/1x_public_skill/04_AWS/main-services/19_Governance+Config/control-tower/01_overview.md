# Control Tower 概要

## 1. サービスの役割

AWS Control Towerは、マルチアカウントAWS環境の設定とガバナンスを自動化するサービスです。ベストプラクティスに基づいた安全で準拠したマルチアカウント環境を簡単に構築・管理できます。

## 2. 主なユースケース

- マルチアカウント環境の初期設定
- ガバナンスとコンプライアンスの自動化
- ベストプラクティスの適用
- アカウントの自動プロビジョニング
- セキュリティポリシーの一元管理
- コンプライアンス監査の自動化

## 3. 他サービスとの関係

- **Organizations**: アカウントの管理
- **Config**: コンプライアンス管理
- **CloudTrail**: 監査ログ
- **IAM**: アクセス制御
- **Service Catalog**: 承認されたリソースの提供
- **Security Hub**: セキュリティの可視化

## 4. 料金のざっくりイメージ

- Control Tower自体は無料
- 関連するサービス（Config、CloudTrail、Security Hubなど）に料金が発生
- **注意**: 管理アカウントとログアーカイブアカウントの料金が発生

## 5. 初心者がまず覚えるべきポイント

- マルチアカウント環境のガバナンスサービス
- ベストプラクティスに基づいた環境の自動設定
- **ランディングゾーン**: 安全で準拠したマルチアカウント環境
- **管理アカウント**: Control Towerを設定するアカウント
- **ログアーカイブアカウント**: ログを保存するアカウント
- **監査アカウント**: 監査用のアカウント
- ガードレール（ポリシー）でコンプライアンスを強制
- アカウントファクトリーでアカウントを自動プロビジョニング
- コンプライアンスダッシュボードで環境の状態を確認
- Organizationsと統合してアカウントを管理

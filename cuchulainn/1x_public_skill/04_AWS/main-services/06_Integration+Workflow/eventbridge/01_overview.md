# EventBridge 概要

## 1. サービスの役割

Amazon EventBridgeは、イベント駆動型アプリケーションを構築するためのサーバーレスイベントバスサービスです。AWSサービス、カスタムアプリケーション、SaaSアプリケーションからのイベントを受信し、ルールに基づいてターゲット（Lambda、SNS、SQSなど）にルーティングします。

## 2. 主なユースケース

- イベント駆動型アプリケーションの構築
- マイクロサービス間の疎結合な通信
- カスタムイベントのルーティング
- スケジュール実行（Cron式）
- SaaSアプリケーション（Salesforce、Zendeskなど）との統合
- イベントのフィルタリングと変換
- ワークフローのオーケストレーション

## 3. 他サービスとの関係

- **Lambda**: イベントのターゲットとして使用
- **SNS, SQS**: イベントの配信先
- **Step Functions**: ワークフローの実行
- **CloudWatch Events**: EventBridgeの前身（現在は統合）
- **S3, EC2, RDS**: AWSサービスからのイベント
- **SaaSアプリケーション**: サードパーティからのイベント

## 4. 料金のざっくりイメージ

- **カスタムイベント**: 最初の100万イベント/月は無料、以降$1.00/100万イベント
- **AWSイベント**: 無料（AWSサービスからのイベント）
- **SaaSイベント**: イベントあたり$0.0025（SaaSアプリケーションからのイベント）
- **注意**: カスタムイベントとSaaSイベントのみ課金

## 5. 初心者がまず覚えるべきポイント

- イベント駆動型アプリケーションのイベントバス
- イベントソース（AWSサービス、カスタムアプリ、SaaSアプリ）からイベントを受信
- ルールでイベントをフィルタリングしてターゲットにルーティング
- ターゲット（Lambda、SNS、SQS、Step Functions、EventBridgeなど）
- スケジュール実行（Cron式で定期的にイベントを発行）
- イベントの変換（入力変換、出力変換）
- イベントアーカイブ（過去のイベントを保存）
- リプレイ機能（過去のイベントを再実行）
- CloudWatch EventsからEventBridgeに統合された

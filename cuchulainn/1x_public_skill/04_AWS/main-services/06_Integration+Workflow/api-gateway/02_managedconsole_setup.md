# API Gateway マネージドコンソールでのセットアップ

## 作成するもの

REST APIを作成し、Lambda関数をバックエンドとして設定します。HTTPエンドポイント経由でLambda関数を呼び出せるようにします。

## セットアップ手順

1. **API Gatewayコンソールを開く**
   - AWSマネージメントコンソールで「API Gateway」を検索して選択

2. **APIを作成**
   - 「APIを作成」をクリック
   - **APIタイプ**: REST APIを選択
   - 「構築」をクリック

3. **APIの設定**
   - **プロトコル**: RESTを選択
   - **新しいAPIを作成**: 新しいAPIを選択
   - **API名**: `my-api` を入力
   - **エンドポイントタイプ**: リージョン（デフォルト）
   - 「APIを作成」をクリック

4. **リソースとメソッドを作成**
   - 「アクション」→「リソースを作成」をクリック
   - **リソース名**: `hello` を入力
   - 「リソースを作成」をクリック
   - 作成したリソースを選択
   - 「アクション」→「メソッドを作成」をクリック
   - **メソッド**: GETを選択
   - 「チェックマーク」をクリック

5. **統合を設定**
   - **統合タイプ**: Lambda関数を選択
   - **Lambda関数**: 既存のLambda関数を選択
   - 「保存」をクリック

6. **APIをデプロイ**
   - 「アクション」→「APIのデプロイ」をクリック
   - **デプロイステージ**: [新しいステージ]を選択
   - **ステージ名**: `prod` を入力
   - 「デプロイ」をクリック

7. **確認**
   - 表示されたURL（例: `https://xxxxx.execute-api.ap-northeast-1.amazonaws.com/prod/hello`）でAPIをテスト

## 補足

- APIのURLでHTTPリクエストを送信できます
- ステージごとに異なる環境（dev、prodなど）を管理できます

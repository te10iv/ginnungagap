# Amazon SQS：まずこれだけ（Lv1）

## このサービスの一言説明
- Amazon SQS は「**マネージドメッセージキューサービス**」を提供するAWSサービス

## ゴール（ここまでできたら合格）
- SQSキューを **作成できる**
- **メッセージを送信・受信できる**
- 「非同期処理にはSQSが必要」と判断できる

## まず覚えること（最小セット）
- **キュー**：メッセージの待ち行列
- **標準キュー**：順序保証なし、最低1回配信、無制限スループット
- **FIFOキュー**：順序保証、厳密に1回配信、3000メッセージ/秒
- **ポーリング**：メッセージ取得（短期・長期）
- **可視性タイムアウト**：処理中メッセージの非表示時間

## できるようになること
- □ キューを作成できる
- □ メッセージを送信できる
- □ メッセージを受信・削除できる
- □ デッドレターキューを設定できる

## まずやること（Hands-on）
- 標準キュー作成
- AWS CLIでメッセージ送信
- メッセージ受信
- メッセージ削除

## 関連するAWSサービス（名前だけ）
- **Lambda**：イベントソース、自動ポーリング
- **SNS**：ファンアウト（SNS → SQS）
- **EC2 / ECS**：ワーカー
- **CloudWatch**：メトリクス監視
- **EventBridge**：イベント駆動

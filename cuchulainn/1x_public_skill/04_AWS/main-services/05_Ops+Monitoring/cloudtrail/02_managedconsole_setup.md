# CloudTrail マネージドコンソールでのセットアップ

## 作成するもの

CloudTrailを有効化し、AWS API呼び出しの監査ログを記録します。セキュリティ監査とコンプライアンスのために使用します。

## セットアップ手順

1. **CloudTrailコンソールを開く**
   - AWSマネージメントコンソールで「CloudTrail」を検索して選択

2. **トレイルを作成**
   - 「トレイルを作成」をクリック

3. **基本設定**
   - **トレイル名**: `my-cloudtrail` を入力
   - **適用範囲**: すべてのリージョンに適用を選択

4. **イベントの選択**
   - **イベントタイプ**: 管理イベントを選択
   - **管理イベント**: すべての管理イベントを記録を選択
   - **データイベント**: なし（デフォルト、必要に応じて有効化）

5. **ストレージの場所**
   - **S3バケット**: 新しいS3バケットを作成または既存のバケットを選択
   - **ログファイルの暗号化**: 無効（デフォルト）または有効

6. **確認と作成**
   - 設定を確認
   - 「次へ」→「トレイルを作成」をクリック

7. **確認**
   - トレイルの状態が「ログ配信中」になっていることを確認

## 補足

- CloudTrailのログはS3バケットに保存されます
- イベント履歴で過去90日間のイベントを確認できます

# CloudWatch 概要

## 1. サービスの役割

Amazon CloudWatchは、AWSリソースとアプリケーションの監視サービスです。メトリクス（CPU使用率、メモリ使用率など）の収集、アラームの設定、ダッシュボードの作成、ログの集約などを行い、システムの健全性を監視します。

## 2. 主なユースケース

- AWSリソース（EC2、RDS、Lambdaなど）のメトリクス監視
- カスタムメトリクスの収集と監視
- アラームの設定と通知（SNS経由）
- ダッシュボードでの可視化
- ログの集約と分析
- 自動スケーリングのトリガー
- アプリケーションのパフォーマンス監視

## 3. 他サービスとの関係

- **EC2, RDS, Lambda**: 各種AWSサービスのメトリクスを自動収集
- **SNS**: アラームの通知先
- **Auto Scaling**: メトリクスに基づくスケーリング
- **CloudWatch Logs**: ログの集約と分析
- **EventBridge**: イベントのルーティング
- **IAM**: メトリクスとログへのアクセス制御

## 4. 料金のざっくりイメージ

- **カスタムメトリクス**: メトリクスあたり$0.30/月（最初の10,000メトリクスは無料）
- **アラーム**: アラームあたり$0.10/月（最初の10個のアラームは無料）
- **ダッシュボード**: ダッシュボードあたり$3/月（最初の3個は無料）
- **APIリクエスト**: $0.01/1,000リクエスト
- **注意**: 標準メトリクス（AWSサービスが自動的に送信するメトリクス）は無料

## 5. 初心者がまず覚えるべきポイント

- AWSリソースの監視サービス（メトリクス、ログ、アラーム）
- 標準メトリクスは自動的に収集される（無料）
- カスタムメトリクスを送信可能（アプリケーションの独自メトリクス）
- アラームで閾値を設定して通知（SNS経由）
- ダッシュボードで複数のメトリクスを可視化
- メトリクスの保持期間（1分、5分、1時間、1日など）
- CloudWatch Logsと統合してログも監視
- Auto Scalingのトリガーとして使用
- メトリクス数学（複数のメトリクスを組み合わせて計算）

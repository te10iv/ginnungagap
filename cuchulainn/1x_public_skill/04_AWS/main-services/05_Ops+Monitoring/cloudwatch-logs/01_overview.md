# CloudWatch Logs 概要

## 1. サービスの役割

Amazon CloudWatch Logsは、ログデータの収集、監視、分析、保存を行うサービスです。AWSリソースやアプリケーションからログを集約し、リアルタイムで監視、検索、分析できます。

## 2. 主なユースケース

- アプリケーションログの集約と分析
- システムログの監視
- エラーログの検索と分析
- ログベースのアラーム設定
- ログの長期保存
- ログストリームのリアルタイム監視
- ログインサイトの分析

## 3. 他サービスとの関係

- **Lambda**: Lambda関数のログを自動的に送信
- **EC2, ECS**: アプリケーションログをCloudWatch Logsに送信
- **VPC Flow Logs**: VPCのネットワークトラフィックログ
- **Route 53**: DNSクエリログ
- **CloudWatch**: ログベースのメトリクスとアラーム
- **EventBridge**: ログイベントのルーティング
- **Kinesis**: ログストリームの処理

## 4. 料金のざっくりイメージ

- **ログの取り込み**: GBあたり$0.50（最初の5GB/月は無料）
- **ログの保存**: GBあたり$0.03/月（最初の5GB/月は無料）
- **ログのデータスキャン**: GBあたり$0.005
- **注意**: ログの量が多いとコストが高くなる

## 5. 初心者がまず覚えるべきポイント

- ログの集約と分析サービス
- ロググループ（論理的なログの集合）とログストリーム（個々のログソース）
- ログインサイト（リアルタイムでログを監視）
- ログフィルター（特定の条件に一致するログを抽出）
- ログベースのメトリクス（ログからメトリクスを生成）
- ログの保持期間（1日〜10年、または永続保存）
- ログのエクスポート（S3、Kinesis Streams、Kinesis Data Firehose）
- CloudWatch Logs Insights（SQLライクなクエリでログを分析）
- 暗号化（KMSで暗号化可能）

# Systems Manager マネージドコンソールでのセットアップ

## 作成するもの

SSM Session Managerを設定し、EC2インスタンスにSSH接続なしでアクセスできるようにします。セキュアなリモートアクセスを実現します。

## セットアップ手順

1. **IAMロールを作成**
   - IAMコンソールで「ロール」を選択
   - 「ロールを作成」をクリック
   - **信頼されたエンティティ**: EC2を選択
   - **ポリシー**: `AmazonSSMManagedInstanceCore` をアタッチ
   - **ロール名**: `SSMInstanceRole` を入力
   - 「ロールを作成」をクリック

2. **EC2インスタンスにIAMロールをアタッチ**
   - EC2コンソールでインスタンスを選択
   - 「アクション」→「セキュリティ」→「IAMロールを変更」をクリック
   - 作成したIAMロールを選択
   - 「更新」をクリック

3. **SSM Agentの確認**
   - 最新のAmazon Linux 2023にはSSM Agentがプリインストールされています

4. **Session Managerで接続**
   - EC2コンソールでインスタンスを選択
   - 「接続」をクリック
   - 「Session Manager」タブを選択
   - 「接続」をクリック

## 補足

- SSM Agentがインスタンスにインストールされている必要があります
- セキュリティグループでSSHポートを開く必要がありません

# Systems Manager（SSM） 概要

## 1. サービスの役割

AWS Systems Manager（SSM）は、EC2インスタンスやオンプレミスサーバーを一元管理するサービスです。パッチ適用、設定管理、インベントリ収集、セッションマネージャー、パラメータストアなどの機能を提供します。

## 2. 主なユースケース

- EC2インスタンスへのSSH接続なしでのアクセス（Session Manager）
- パッチの自動適用
- 設定の一元管理（State Manager）
- インベントリの収集と管理
- パラメータの安全な保存（Parameter Store）
- コマンドの一括実行（Run Command）
- メンテナンスウィンドウの管理

## 3. 他サービスとの関係

- **EC2**: EC2インスタンスの管理対象
- **IAM**: SSMへのアクセス制御
- **CloudWatch**: SSMのメトリクスとログ
- **S3**: SSMドキュメントやログの保存
- **VPC**: VPC内のリソースにアクセス
- **Lambda**: SSMドキュメントの実行

## 4. 料金のざっくりイメージ

- **SSM Agent**: 無料（EC2インスタンスにインストール済み）
- **Session Manager**: 無料
- **Parameter Store**: 標準パラメータは無料、高度なパラメータは$0.05/月
- **Patch Manager**: 無料（パッチのダウンロード料金は別途）
- **注意**: ほとんどの機能が無料

## 5. 初心者がまず覚えるべきポイント

- EC2インスタンスの一元管理サービス
- **Session Manager**: SSH接続なしでEC2インスタンスにアクセス（セキュア）
- **Parameter Store**: パスワードやAPIキーなどの機密情報を安全に保存
- **Patch Manager**: パッチの自動適用とコンプライアンス管理
- **Run Command**: 複数のインスタンスにコマンドを一括実行
- **State Manager**: インスタンスの設定を自動的に維持
- **Inventory**: インスタンスのソフトウェアと設定のインベントリを収集
- SSM Agentがインスタンスにインストールされている必要がある
- IAMロールでSSMへのアクセス権限を付与

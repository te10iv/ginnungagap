# Auto Scaling マネージドコンソールでのセットアップ

## 作成するもの

Auto Scalingグループを作成し、EC2インスタンス数を自動的にスケールします。CPU使用率に基づいて、需要に応じてインスタンス数を増減させます。

## セットアップ手順

1. **Launch Templateを作成**
   - EC2コンソールで「起動テンプレート」を選択
   - 「起動テンプレートを作成」をクリック
   - **名前**: `my-launch-template` を入力
   - **AMI**: Amazon Linux 2023を選択
   - **インスタンスタイプ**: t3.microを選択
   - **キーペア**: 既存のキーペアを選択
   - **セキュリティグループ**: 既存のセキュリティグループを選択
   - 「起動テンプレートを作成」をクリック

2. **Auto Scalingグループを作成**
   - EC2コンソールで「Auto Scalingグループ」を選択
   - 「Auto Scalingグループを作成」をクリック

3. **基本設定**
   - **名前**: `my-asg` を入力
   - **起動テンプレート**: 作成した起動テンプレートを選択
   - **バージョン**: 最新を選択

4. **ネットワークを設定**
   - **VPC**: 作成済みのVPCを選択
   - **サブネット**: 複数のAZのサブネットを選択

5. **容量を設定**
   - **希望する容量**: 2
   - **最小サイズ**: 1
   - **最大サイズ**: 4

6. **スケーリングポリシーを設定**
   - 「ターゲット追跡スケーリングポリシーを追加」をクリック
   - **メトリクス**: 平均CPU使用率を選択
   - **ターゲット値**: 50%
   - 「次へ」をクリック

7. **確認と作成**
   - 設定を確認
   - 「Auto Scalingグループを作成」をクリック

## 補足

- ターゲット追跡スケーリングポリシーでCPU使用率を50%に維持します
- インスタンス数は最小1、最大4の範囲で自動調整されます

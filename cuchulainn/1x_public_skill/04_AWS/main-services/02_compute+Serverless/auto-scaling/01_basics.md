# AWS Auto Scaling：まずこれだけ（Lv1）

## このサービスの一言説明
- AWS Auto Scaling は「**EC2インスタンスを負荷に応じて自動的に増減させる**」サービス

## ゴール（ここまでできたら合格）
- Auto Scaling Groupを **作成できる**
- **スケーリングポリシーの基本を説明できる**
- 「このシステムにはAuto Scalingが必要」と判断できる

## まず覚えること（最小セット）
- **Auto Scaling Group（ASG）**：EC2インスタンスのグループ管理単位
- **起動テンプレート**：EC2の設定テンプレート（AMI、インスタンスタイプ等）
- **希望容量・最小・最大**：インスタンス台数の設定（例：最小2、希望3、最大10）
- **スケーリングポリシー**：増減のルール（CPU 70%超えたら追加等）
- **ヘルスチェック**：異常インスタンスの自動置き換え

## できるようになること
- □ マネジメントコンソールでAuto Scaling Groupを作成できる
- □ 起動テンプレートを作成できる
- □ スケーリングポリシーを設定できる（ターゲット追跡）
- □ ALBと連携できる

## まずやること（Hands-on）
- 起動テンプレート作成（AMI、t3.micro、User Data）
- Auto Scaling Group作成（最小2、希望2、最大4）
- ターゲット追跡スケーリング設定（CPU 70%）
- ALBのターゲットグループに自動登録設定
- 負荷テストでスケールアウト確認

## 関連するAWSサービス（名前だけ）
- **EC2**：Auto Scalingで管理するインスタンス
- **起動テンプレート**：EC2設定テンプレート
- **ALB / NLB**：ターゲットグループに自動登録
- **CloudWatch**：メトリクス監視、スケーリングトリガー
- **VPC / Subnet**：インスタンス配置場所

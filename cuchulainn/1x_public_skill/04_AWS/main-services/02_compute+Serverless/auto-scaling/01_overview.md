# Auto Scaling 概要

## 1. サービスの役割

Auto Scalingは、アプリケーションの需要に応じてコンピューティングリソースを自動的にスケール（増減）するサービスです。EC2インスタンス、ECSタスク、DynamoDBテーブル、Spot Fleetなどのリソース数を自動的に調整し、コスト最適化と高可用性を実現します。

## 2. 主なユースケース

- トラフィックの変動に応じたEC2インスタンス数の自動調整
- コスト最適化（需要が少ない時間帯にインスタンス数を削減）
- 高可用性の確保（複数AZにインスタンスを分散）
- ECSタスク数の自動スケーリング
- DynamoDBの読み書きキャパシティの自動調整
- バッチ処理ジョブの並列実行数の調整

## 3. 他サービスとの関係

- **EC2**: EC2インスタンスの自動スケーリング
- **Launch Template / Launch Configuration**: スケールアウト時に起動するインスタンスの設定
- **CloudWatch**: メトリクス（CPU使用率など）に基づくスケーリング
- **ELB**: ロードバランサーのターゲットとしてインスタンスを登録
- **ECS**: ECSタスクの自動スケーリング
- **DynamoDB**: テーブルの読み書きキャパシティの自動調整
- **SNS**: スケーリングイベントの通知

## 4. 料金のざっくりイメージ

- Auto Scaling自体は無料
- スケールアウトで起動したリソース（EC2、ECSタスクなど）の料金が発生
- CloudWatchアラームの料金（詳細モニタリングを使用する場合）

## 5. 初心者がまず覚えるべきポイント

- **スケールアウト**: 需要増加時にリソースを追加
- **スケールイン**: 需要減少時にリソースを削減
- **最小サイズ**: 常に維持するリソース数
- **最大サイズ**: スケールアウトの上限
- **希望サイズ**: 現在の目標リソース数
- **スケーリングポリシー**: スケーリングのトリガー（CPU使用率、リクエスト数など）
- **クールダウン期間**: スケーリングアクション後の待機時間
- **ヘルスチェック**: 異常なインスタンスを自動的に置き換え
- 複数AZに分散して高可用性を確保

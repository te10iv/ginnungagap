# EC2 概要

## 1. サービスの役割

Amazon EC2（Elastic Compute Cloud）は、AWSクラウド内で仮想サーバー（インスタンス）を提供するサービスです。オンデマンドで仮想マシンを起動・停止・管理でき、様々なインスタンスタイプ、OS、ストレージオプションから選択できます。

## 2. 主なユースケース

- Webアプリケーションサーバーの構築
- データベースサーバーの構築
- バッチ処理やジョブの実行
- 開発・テスト環境の構築
- コンテナホスト（ECS、EKS）としての利用
- オンプレミス環境からの移行

## 3. 他サービスとの関係

- **VPC**: EC2インスタンスを配置するネットワーク環境
- **EBS**: インスタンスのブロックストレージ
- **ELB**: 複数のEC2インスタンスへのロードバランシング
- **Auto Scaling**: インスタンス数の自動スケーリング
- **IAM**: インスタンスへのアクセス制御
- **Security Group**: ネットワークレベルのセキュリティ制御
- **CloudWatch**: メトリクスとログの監視
- **SSM**: インスタンスの管理とパッチ適用

## 4. 料金のざっくりイメージ

- **オンデマンド**: インスタンスタイプにより異なる（例: t3.microで約$0.0104/時間 = 約$7.5/月）
- **Reserved Instance**: 1年契約で約40%割引、3年契約で約60%割引
- **Spot Instance**: オンデマンドの最大90%割引（中断される可能性あり）
- **Savings Plans**: 柔軟な料金プランで約17-72%割引
- **データ転送料金**: リージョン外への転送で料金発生

## 5. 初心者がまず覚えるべきポイント

- インスタンスタイプ（t3、m5、c5など）でCPU、メモリ、ネットワーク性能が決まる
- AMI（Amazon Machine Image）でOSとソフトウェアを選択
- キーペアでSSH接続の認証を行う
- セキュリティグループでファイアウォール設定
- EBSボリュームでストレージを管理
- パブリックIPとプライベートIPの違いを理解
- インスタンスストア（一時ストレージ）とEBS（永続ストレージ）の違い
- 停止と終了の違い（停止は課金継続、終了は課金停止）

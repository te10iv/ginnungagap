# Lambda 概要

## 1. サービスの役割

AWS Lambdaは、サーバーの管理なしでコードを実行できるサーバーレスコンピューティングサービスです。イベントに応じて自動的にコードを実行し、使用した分だけ課金されます。インフラストラクチャの管理が不要で、自動的にスケーリングされます。

## 2. 主なユースケース

- イベント駆動型アプリケーション（S3へのファイルアップロード、DynamoDBの変更など）
- APIバックエンド（API Gatewayと連携）
- データ処理と変換（ETL処理、画像処理など）
- スケジュール実行（CloudWatch Eventsと連携）
- マイクロサービスアーキテクチャ
- サーバーレスアプリケーションの構築

## 3. 他サービスとの関係

- **API Gateway**: REST APIやHTTP APIのバックエンドとして使用
- **S3**: オブジェクトのアップロード・変更イベントでトリガー
- **DynamoDB**: テーブルの変更イベントでトリガー
- **EventBridge**: カスタムイベントでトリガー
- **SNS, SQS**: メッセージを受信してトリガー
- **CloudWatch Events**: スケジュール実行でトリガー
- **Step Functions**: ワークフローの一部として実行
- **VPC**: VPC内のリソースにアクセスする場合に設定

## 4. 料金のざっくりイメージ

- **リクエスト**: 最初の100万リクエストまで無料、以降$0.20/100万リクエスト
- **実行時間**: GB秒あたり$0.0000166667（メモリサイズと実行時間に基づく）
- **無料利用枠**: 毎月100万リクエスト、400,000 GB秒
- **注意**: 実行時間が短い、リクエスト数が少ない場合は非常に安価

## 5. 初心者がまず覚えるべきポイント

- サーバー管理が不要（インフラストラクチャの管理はAWSが行う）
- イベント駆動で自動実行
- 自動スケーリング（同時実行数の上限あり）
- サポート言語: Node.js、Python、Java、C#、Go、Ruby、.NET Coreなど
- 最大実行時間: 15分（2021年10月から）
- メモリ: 128MB〜10GB（CPUはメモリに比例）
- コールドスタート: 初回実行時に遅延が発生する可能性
- 環境変数で設定を管理
- IAMロールで他のAWSサービスへのアクセス権限を制御

あなたは AWS Lambda を実務で運用・開発してきたプロです。
来月から Lambda を使う案件に入る初学者が、
「初日〜1週間」で最低限戦力化するための
**Lambda 実務チェックリスト**を Markdown で作成してください。

## 目的
- 初日に迷わない
- 1週間で「調査・修正・デプロイ・運用の基本動作」ができる
- 事故を起こしやすいポイントを事前に潰す

## 前提
- 学習者は Lambda 初学者（AWSは触ったことがある程度）
- 重点は「実務で頻出の流れ」と「トラブル対応」
- 机上知識ではなく、具体的な手順・確認項目に寄せる

## 出力形式（必須）
- Markdown
- 見出し構造は以下に従う

# Lambda 実務チェックリスト（初日〜1週間）

  - 完了条件：

…同様

…同様

## 付録：最短で役立つコマンド・確認場所
- CloudWatch Logs の見方
- Lambda 設定で見るべき場所
- IAM で見るべき場所
- 可能なら AWS CLI 例（任意）

## チェックリストに必ず含める項目（重要）
- どの Lambda がどのトリガーで動いているか把握する
- ログの場所・追い方（CloudWatch Logs）
- タイムアウト／メモリ／リトライ／DLQ の考え方
- 失敗時の調査の流れ（どこを見るか）
- デプロイ手段（Console / SAM / CI/CD のいずれか）把握
- 環境変数・Secrets の扱い（直書き禁止など）
- VPC 接続有無の確認と注意点
- 権限（実行ロール）と最小権限の基本
- 変更前後の影響範囲とロールバック観点

## 注意
- 「案件による」逃げは最小限にし、汎用的に使える形にする
- 初学者が明日から使えるレベルの具体性を優先

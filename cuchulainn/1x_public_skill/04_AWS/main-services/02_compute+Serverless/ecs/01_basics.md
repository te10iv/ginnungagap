# Amazon ECS：まずこれだけ（Lv1）

## このサービスの一言説明
- Amazon ECS は「**Dockerコンテナを実行・管理するオーケストレーションサービス**」

## ゴール（ここまでできたら合格）
- ECSクラスター・サービスを **作成できる**
- **タスク定義の役割を説明できる**
- 「コンテナアプリにはECSが必要」と判断できる

## まず覚えること（最小セット）
- **ECSクラスター**：コンテナを実行するグループ
- **タスク定義**：Dockerコンテナの設定（イメージ、CPU、メモリ等）
- **タスク**：タスク定義の実行インスタンス
- **サービス**：タスクの数を維持し、ALBと連携
- **起動タイプ**：Fargate（サーバーレス）or EC2（自前管理）

## できるようになること
- □ マネジメントコンソールでECSクラスターを作成できる
- □ タスク定義を作成できる（ECRイメージ指定）
- □ サービスを作成してタスクを起動できる
- □ ALBと連携できる

## まずやること（Hands-on）
- ECSクラスター作成（Fargate起動タイプ）
- タスク定義作成（ECRイメージ、CPU 256、メモリ 512）
- サービス作成（希望タスク数2、ALB連携）
- ALBのDNS名でアクセス確認

## 関連するAWSサービス（名前だけ）
- **ECR**：コンテナイメージ保存
- **Fargate**：サーバーレスコンピューティング
- **ALB / NLB**：ロードバランサー
- **CloudWatch Logs**：コンテナログ
- **IAM**：タスク実行ロール・タスクロール
- **VPC / Subnet**：タスク配置場所

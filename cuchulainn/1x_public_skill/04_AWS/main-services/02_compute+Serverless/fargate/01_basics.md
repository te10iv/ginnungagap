# AWS Fargate：まずこれだけ（Lv1）

## このサービスの一言説明
- AWS Fargate は「**コンテナをサーバーレスで実行する**」サービス

## ゴール（ここまでできたら合格）
- Fargateで **コンテナを起動できる**
- **EC2起動タイプとの違いを説明できる**
- 「サーバーレスコンテナにはFargateが必要」と判断できる

## まず覚えること（最小セット）
- **Fargate**：ECSの起動タイプの一つ（サーバーレス）
- **サーバー管理不要**：EC2インスタンス管理が不要
- **従量課金**：vCPU時間 + メモリ時間で課金
- **タスクレベル分離**：各タスクが独立した環境
- **ECS or EKS**：FargateはECS/EKSの両方で使用可能

## できるようになること
- □ ECSでFargate起動タイプを選択できる
- □ タスク定義でCPU/メモリを設定できる
- □ Fargateタスクを起動できる
- □ EC2起動タイプとの使い分けができる

## まずやること（Hands-on）
- ECSクラスター作成（Fargate起動タイプ）
- タスク定義作成（Fargate、0.25vCPU、0.5GB）
- サービス作成（希望タスク数1）
- タスク起動確認、コンテナログ確認

## 関連するAWSサービス（名前だけ）
- **ECS**：Fargateでタスク実行
- **EKS**：KubernetesでもFargate使用可能
- **ECR**：コンテナイメージ保存
- **ALB / NLB**：ロードバランサー
- **CloudWatch Logs**：コンテナログ
- **VPC / Subnet**：タスク配置場所

# Fargate マネージドコンソールでのセットアップ

## 作成するもの

ECS on Fargateでコンテナを実行します。サーバー管理不要で、コンテナを直接実行できます。

## セットアップ手順

1. **ECSクラスターを作成**
   - ECSコンソールで「クラスターを作成」をクリック
   - **クラスター名**: `my-fargate-cluster` を入力
   - **インフラストラクチャ**: AWS Fargateを選択
   - 「作成」をクリック

2. **タスク定義を作成**
   - 「タスク定義」を選択
   - 「新しいタスク定義の作成」をクリック
   - **タスク定義ファミリー**: `my-fargate-task` を入力
   - **起動タイプ**: Fargateを選択
   - **タスクサイズ**: 0.25 vCPU、0.5 GBメモリを選択（最小構成）
   - **コンテナ**: 「コンテナを追加」をクリック
     - **コンテナ名**: `my-fargate-container` を入力
     - **イメージURI**: `nginx:latest` を入力
     - **ポートマッピング**: 80（HTTP）を追加
   - 「作成」をクリック

3. **タスクを実行**
   - クラスターを選択
   - 「タスク」タブで「新しいタスクの実行」をクリック
   - **起動タイプ**: Fargateを選択
   - **タスク定義**: 作成したタスク定義を選択
   - **VPC**: 作成済みのVPCを選択
   - **サブネット**: パブリックサブネットを選択
   - **セキュリティグループ**: HTTPポート80を許可するセキュリティグループを選択
   - 「タスクを作成」をクリック

4. **確認**
   - タスクの状態が「実行中」になることを確認

## 補足

- Fargateはサーバーレスでコンテナを実行します（EC2インスタンスの管理不要）
- タスクごとに課金されます（実行中のみ）
- サービスとして実行する場合は、ECSサービスの作成が必要です

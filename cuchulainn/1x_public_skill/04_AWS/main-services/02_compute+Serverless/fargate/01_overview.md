# Fargate 概要

## 1. サービスの役割

AWS Fargateは、サーバーレスでコンテナを実行するコンピューティングエンジンです。EC2インスタンスの管理が不要で、コンテナを直接実行できます。ECSやEKSと組み合わせて使用し、インフラストラクチャの管理から解放されます。

## 2. 主なユースケース

- サーバー管理不要でコンテナを実行したい場合
- マイクロサービスアーキテクチャの構築
- コンテナのオーケストレーションが必要だが、インフラ管理はしたくない場合
- 小規模〜中規模のコンテナワークロード
- CI/CDパイプラインでのコンテナ実行
- バッチ処理ジョブの実行

## 3. 他サービスとの関係

- **ECS**: ECS on Fargateとしてコンテナを実行
- **EKS**: EKS on FargateとしてKubernetes Podを実行
- **ECR**: コンテナイメージを取得
- **ALB/NLB**: ロードバランシングと統合
- **VPC**: タスクをVPC内に配置
- **CloudWatch**: ログとメトリクスの監視
- **IAM**: タスクのアクセス制御

## 4. 料金のざっくりイメージ

- **vCPU**: 約$0.04048/時間（1vCPUあたり）
- **メモリ**: 約$0.004445/GB/時間
- **例**: 1vCPU + 2GBメモリ = 約$0.04937/時間（約$35/月）
- **注意**: EC2より高価だが、サーバー管理が不要

## 5. 初心者がまず覚えるべきポイント

- サーバーレスでコンテナを実行（EC2インスタンスの管理不要）
- ECS on FargateとEKS on Fargateの2つのモードがある
- vCPUとメモリを指定してタスクを起動
- タスクごとに課金（実行中のみ課金）
- EC2より高価だが、運用コストが削減できる
- スケーリングが自動的で簡単
- VPC内にタスクを配置可能
- タスク実行ロールでECRへのアクセス権限を設定

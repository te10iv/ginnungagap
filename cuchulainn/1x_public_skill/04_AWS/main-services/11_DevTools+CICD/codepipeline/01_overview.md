# CodePipeline 概要

## 1. サービスの役割

AWS CodePipelineは、CI/CDパイプラインを構築・管理するサービスです。ソースコードの取得、ビルド、テスト、デプロイメントを自動化し、リリースプロセスを効率化します。

## 2. 主なユースケース

- CI/CDパイプラインの構築
- 継続的インテグレーション（CI）
- 継続的デプロイメント（CD）
- マルチ環境でのデプロイメント（dev、staging、prod）
- 自動テストとデプロイメント
- リリースプロセスの自動化

## 3. 他サービスとの関係

- **CodeCommit, GitHub, Bitbucket**: ソースコードの取得
- **CodeBuild**: ビルドとテストの実行
- **CodeDeploy**: アプリケーションのデプロイ
- **S3**: パイプライン成果物の保存
- **Lambda**: カスタムアクションの実行
- **CloudWatch**: パイプラインのメトリクスとログ
- **EventBridge**: パイプラインイベントの通知

## 4. 料金のざっくりイメージ

- **パイプライン**: パイプラインあたり$1/月（最初の1パイプライン/月は無料）
- **アクティブな実行**: 実行あたり$0.002（最初の500実行/月は無料）
- **注意**: パイプライン数と実行数に応じて課金

## 5. 初心者がまず覚えるべきポイント

- CI/CDパイプラインサービス（リリースプロセスの自動化）
- パイプライン（ソース → ビルド → テスト → デプロイの流れ）
- ステージ（パイプラインの各段階）
- アクション（各ステージで実行する処理）
- アーティファクト（ステージ間で受け渡すデータ）
- 手動承認（デプロイ前に承認を要求）
- 並列実行（複数のアクションを並列実行）
- 条件付き実行（条件に基づいてステージをスキップ）
- パイプラインの通知（SNS経由）
- パイプラインの可視化（コンソールでパイプラインの状態を確認）

# CodePipeline マネージドコンソールでのセットアップ

## 作成するもの

CodePipelineパイプラインを作成し、ソース取得→ビルド→デプロイのCI/CDパイプラインを構築します。

## セットアップ手順

1. **CodePipelineコンソールを開く**
   - AWSマネージメントコンソールで「CodePipeline」を検索して選択

2. **パイプラインを作成**
   - 「パイプラインを作成」をクリック

3. **パイプライン設定**
   - **パイプライン名**: `my-pipeline` を入力
   - **サービスロール**: 新しいサービスロールを作成

4. **ソースステージを設定**
   - **ソースプロバイダー**: AWS CodeCommitを選択
   - **リポジトリ名**: 既存のCodeCommitリポジトリを選択
   - **ブランチ名**: mainを選択
   - **変更の検出**: CloudWatch Eventsを選択

5. **ビルドステージを追加**
   - 「ビルドステージを追加」をクリック
   - **ビルドプロバイダー**: AWS CodeBuildを選択
   - **プロジェクト名**: 既存のCodeBuildプロジェクトを選択

6. **デプロイステージを追加（オプション）**
   - 「デプロイステージを追加」をクリック
   - **デプロイプロバイダー**: AWS CodeDeployを選択
   - **アプリケーション名**: 既存のCodeDeployアプリケーションを選択
   - **デプロイメントグループ**: 既存のデプロイメントグループを選択

7. **確認と作成**
   - 設定を確認
   - 「パイプラインを作成」をクリック

8. **パイプラインの実行**
   - パイプラインが自動的に実行されます
   - 各ステージの進行状況を確認

## 補足

- ソースコードの変更を検出すると自動的にパイプラインが実行されます
- 各ステージの成功/失敗を確認できます

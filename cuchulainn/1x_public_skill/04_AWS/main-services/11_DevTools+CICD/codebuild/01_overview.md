# CodeBuild 概要

## 1. サービスの役割

AWS CodeBuildは、ソースコードをコンパイル、テスト、パッケージ化する完全マネージドのビルドサービスです。ビルドサーバーの管理が不要で、スケーラブルで高速なビルド環境を提供します。

## 2. 主なユースケース

- アプリケーションのビルドとコンパイル
- ユニットテストと統合テストの実行
- Dockerイメージのビルド
- デプロイメントパッケージの作成
- CI/CDパイプラインのビルドステージ
- マルチ環境でのビルド（dev、staging、prod）

## 3. 他サービスとの関係

- **CodeCommit, GitHub, Bitbucket**: ソースコードの取得
- **ECR, Docker Hub**: Dockerイメージのプッシュ
- **S3**: ビルド成果物の保存
- **CodePipeline**: CI/CDパイプラインの構築
- **CloudWatch**: ビルドログとメトリクス
- **IAM**: ビルド環境へのアクセス制御

## 4. 料金のざっくりイメージ

- **ビルド時間**: 分あたり$0.005（Linux、最初の100分/月は無料）
- **注意**: ビルド時間に応じて課金（使用した分だけ課金）

## 5. 初心者がまず覚えるべきポイント

- サーバーレスビルドサービス（ビルドサーバーの管理不要）
- ビルドスペック（buildspec.yml）でビルド手順を定義
- 複数のビルド環境（Linux、Windows、ARM）
- カスタムビルド環境（Dockerイメージを使用）
- ビルドキャッシュでビルド時間を短縮
- 並列ビルド（複数のビルドを同時実行）
- ビルドログをCloudWatch Logsに送信
- ビルド成果物をS3に保存
- ビルド通知（SNS経由）
- ビルドのタイムアウト設定

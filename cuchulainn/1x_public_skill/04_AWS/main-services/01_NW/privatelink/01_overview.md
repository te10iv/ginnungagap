# PrivateLink 概要

## 1. サービスの役割

AWS PrivateLinkは、VPC内のリソースが他のVPCやAWSサービス、オンプレミス環境のサービスにプライベートに接続するためのサービスです。インターネット経由せずに、AWSのプライベートネットワーク内で安全にサービスを公開・利用できます。

## 2. 主なユースケース

- VPC間でのプライベートなサービス接続（マルチテナント環境など）
- AWSマネージドサービス（S3、DynamoDBなど）へのプライベート接続
- サードパーティのSaaSサービスへのプライベート接続
- オンプレミス環境からAWSサービスへのプライベート接続
- セキュリティ要件が厳しい環境でのサービス公開

## 3. 他サービスとの関係

- **VPC**: VPC内にVPC Endpoint Serviceを作成
- **VPC Endpoint**: サービスを利用する側でVPC Endpointを作成
- **NLB**: VPC Endpoint Serviceのバックエンドとして使用
- **Route53**: プライベートホストゾーンでDNS解決
- **EC2, Lambda**: サービスを提供する側のリソース

## 4. 料金のざっくりイメージ

- VPC Endpoint: 約$0.01/時間（約$7/月）
- データ転送料金: GBあたり$0.01（同一リージョン内）
- NLBの料金も別途発生

## 5. 初心者がまず覚えるべきポイント

- インターネット経由しないため、セキュリティが向上
- VPC PeeringやTransit Gatewayとは異なり、ルートテーブルの設定が不要
- サービス提供側と利用側の両方で設定が必要
- 複数のVPCから同じサービスにアクセス可能
- セキュリティグループでアクセス制御が可能

# Route Table マネージドコンソールでのセットアップ

## 作成するもの

パブリックサブネット用のカスタムルートテーブルを作成し、インターネットゲートウェイへのルートを追加します。これにより、パブリックサブネットからインターネットにアクセスできるようになります。

## セットアップ手順

1. **ルートテーブルコンソールを開く**
   - VPCコンソールで「ルートテーブル」を選択

2. **ルートテーブルを作成**
   - 「ルートテーブルを作成」をクリック
   - **名前タグ**: `public-route-table` を入力
   - **VPC**: 作成済みのVPCを選択
   - 「ルートテーブルを作成」をクリック

3. **ルートを追加**
   - 作成したルートテーブルを選択
   - 「ルート」タブで「ルートを編集」をクリック
   - 「ルートを追加」をクリック
   - **送信先**: `0.0.0.0/0` を入力
   - **ターゲット**: インターネットゲートウェイを選択（事前に作成が必要）
   - 「変更を保存」をクリック

4. **サブネットを関連付け**
   - 「サブネットの関連付け」タブで「サブネットの関連付けを編集」をクリック
   - パブリックサブネットにチェックを入れる
   - 「変更を保存」をクリック

## 補足

- デフォルトルートテーブルにはローカルルート（VPC内通信）が自動的に追加されています
- インターネットゲートウェイは事前に作成し、VPCにアタッチしておく必要があります

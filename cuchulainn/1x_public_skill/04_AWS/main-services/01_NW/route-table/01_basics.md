# Route Table：まずこれだけ（Lv1）

## このサービスの一言説明
- Route Table は「**サブネット内の通信がどこに向かうかを決める道順表**」を提供するAWSの構成要素

## ゴール（ここまでできたら合格）
- Route Tableを **作成してサブネットに関連付けできる**
- **ルート設定の意味を説明できる**
- 「この通信はどのルートを通るか」と判断できる

## まず覚えること（最小セット）
- **Route Table（ルートテーブル）**：通信の送信先を決めるルール集
- **ルート（Route）**：宛先CIDR → ターゲット の組み合わせ
- **ターゲット**：IGW、NAT Gateway、VPC Peering、ローカル など
- **サブネット関連付け**：サブネットごとに1つのルートテーブルを関連付け
- **メインルートテーブル**：VPC作成時に自動作成されるデフォルトのルートテーブル

## できるようになること
- □ マネジメントコンソールでルートテーブルを作成できる
- □ ルート（宛先→ターゲット）を追加できる
- □ サブネットとルートテーブルを関連付けできる
- □ 通信の経路を説明できる

## まずやること（Hands-on）
- マネコンでルートテーブルを1つ作る
- 0.0.0.0/0 → IGW のルートを追加
- Publicサブネットに関連付け
- EC2からインターネット疎通確認（curl）

## 関連するAWSサービス（名前だけ）
- **VPC**：ルートテーブルが属するネットワーク
- **Subnet**：ルートテーブルと関連付けられる
- **Internet Gateway**：インターネット向けルートのターゲット
- **NAT Gateway**：Privateサブネットのインターネット向けルートのターゲット
- **VPC Peering / Transit Gateway**：VPC間通信のターゲット

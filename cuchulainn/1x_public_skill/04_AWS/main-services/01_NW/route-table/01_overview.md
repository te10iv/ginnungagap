# Route Table 概要

## 1. サービスの役割

ルートテーブルは、サブネット内のトラフィックのルーティングを制御するルールの集合です。各サブネットにはルートテーブルが関連付けられ、パケットの送信先に応じて適切なゲートウェイやターゲットに転送されます。

## 2. 主なユースケース

- パブリックサブネットからインターネットへのルーティング（IGW経由）
- プライベートサブネットからインターネットへのアウトバウンド接続（NAT Gateway経由）
- VPC間の通信（VPC Peering、Transit Gateway経由）
- オンプレミス環境への接続（VPN、Direct Connect経由）
- VPC Endpoint経由でのAWSサービスへのプライベート接続

## 3. 他サービスとの関係

- **Subnet**: ルートテーブルはサブネットに関連付けられる
- **Internet Gateway**: インターネットへのルーティング
- **NAT Gateway**: プライベートサブネットからのアウトバウンド接続
- **VPC Peering**: 他のVPCへのルーティング
- **Transit Gateway**: 複数VPC間のルーティング
- **VPC Endpoint**: AWSサービスへのプライベート接続

## 4. 料金のざっくりイメージ

- ルートテーブル自体は無料
- 関連するゲートウェイ（NAT Gateway、Transit Gatewayなど）に料金が発生

## 5. 初心者がまず覚えるべきポイント

- 各VPCにはデフォルトルートテーブルが存在する
- カスタムルートテーブルを作成してサブネットに関連付け可能
- ローカルルート（VPC内通信）は常に存在し、削除不可
- より具体的なルート（小さいCIDR）が優先される
- パブリックサブネットには0.0.0.0/0 → IGWのルートが必要
- プライベートサブネットには0.0.0.0/0 → NAT Gatewayのルートを設定可能

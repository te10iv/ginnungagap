# サブネット（Public / Private） 概要

## 1. サービスの役割

サブネットは、VPC内のIPアドレス範囲を分割した論理的なネットワークセグメントです。パブリックサブネットとプライベートサブネットに分類され、それぞれ異なる用途で使用されます。各サブネットは単一のアベイラビリティゾーン（AZ）に配置されます。

## 2. 主なユースケース

- **パブリックサブネット**: インターネットから直接アクセス可能なリソース（Webサーバー、ロードバランサーなど）の配置
- **プライベートサブネット**: インターネットから直接アクセスできないリソース（データベース、アプリケーションサーバーなど）の配置
- マルチAZ構成による高可用性の実現
- セキュリティ層の分離（DMZ構成など）

## 3. 他サービスとの関係

- **VPC**: サブネットはVPC内に作成される
- **Route Table**: サブネットにルートテーブルを関連付けてトラフィックを制御
- **Internet Gateway**: パブリックサブネットからインターネットへの接続
- **NAT Gateway**: プライベートサブネットからインターネットへのアウトバウンド接続
- **EC2, RDS, ELB**: サブネット内に配置される各種AWSリソース

## 4. 料金のざっくりイメージ

- サブネット自体は無料
- サブネット内で使用するリソース（EC2、RDSなど）の料金が発生
- データ転送料金が発生する場合あり

## 5. 初心者がまず覚えるべきポイント

- サブネットは単一のAZにのみ配置可能（複数AZにまたがれない）
- パブリックサブネット: IGWへのルートがある = インターネットアクセス可能
- プライベートサブネット: IGWへのルートがない = インターネットアクセス不可（NAT Gateway経由でアウトバウンドのみ可能）
- 高可用性のため、各AZにパブリック/プライベートサブネットを配置するのがベストプラクティス
- CIDRブロックは重複してはいけない（例: 10.0.1.0/24 と 10.0.2.0/24）

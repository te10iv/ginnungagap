# Internet Gateway (IGW)：まずこれだけ（Lv1）

## このサービスの一言説明
- Internet Gateway は「**VPCとインターネットを接続するゲート**」を提供するAWSサービス

## ゴール（ここまでできたら合格）
- IGWを **作成してVPCにアタッチできる**
- **IGWがないとインターネット接続できない理由を説明できる**
- 「このVPCにはIGWが必要」と判断できる

## まず覚えること（最小セット）
- **Internet Gateway (IGW)**：VPCとインターネット間の通信を可能にするゲート
- **アタッチ**：IGWをVPCに接続すること（1VPCに1IGW）
- **パブリックIP**：IGWを通じてインターネット通信するにはパブリックIPが必要
- **ルートテーブル**：0.0.0.0/0 → IGW のルートを設定
- **水平スケール**：IGW自体は自動でスケール（帯域制限なし）

## できるようになること
- □ マネジメントコンソールでIGWを作成できる
- □ IGWをVPCにアタッチできる
- □ ルートテーブルにIGW向けのルートを追加できる
- □ EC2からインターネット疎通確認ができる

## まずやること（Hands-on）
- マネコンでIGWを1つ作る
- 作成したIGWをVPCにアタッチ
- ルートテーブルで 0.0.0.0/0 → IGW を設定
- PublicサブネットのEC2から curl でインターネット疎通確認

## 関連するAWSサービス（名前だけ）
- **VPC**：IGWをアタッチする親リソース
- **Route Table**：IGWへのルートを設定
- **Subnet（Public）**：IGW経由でインターネット接続
- **Elastic IP**：固定パブリックIP（IGW経由で通信）
- **NAT Gateway**：PrivateサブネットのインターネットアクセスにIGWが必要

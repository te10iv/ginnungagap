# Internet Gateway（IGW） 概要

## 1. サービスの役割

Internet Gateway（IGW）は、VPCとインターネット間の双方向通信を可能にするマネージドゲートウェイです。パブリックサブネット内のリソースがインターネットにアクセスしたり、インターネットからアクセスされたりする際の出入り口となります。

## 2. 主なユースケース

- パブリックサブネット内のEC2インスタンスからインターネットへのアクセス
- インターネットからパブリックIPを持つリソースへのアクセス
- パブリックロードバランサー（ALB/NLB）へのインターネットからのアクセス
- ソフトウェアのアップデートやパッケージのダウンロード

## 3. 他サービスとの関係

- **VPC**: IGWはVPCにアタッチされる
- **Route Table**: パブリックサブネットのルートテーブルに0.0.0.0/0 → IGWのルートを設定
- **EC2**: パブリックIPを持つEC2インスタンスがIGW経由でインターネットにアクセス
- **ELB**: パブリックロードバランサーがIGW経由でインターネットからアクセス可能
- **NAT Gateway**: プライベートサブネットからのアウトバウンド接続には使用しない（NAT Gatewayを使用）

## 4. 料金のざっくりイメージ

- IGW自体は無料
- データ転送料金が発生（アウトバウンド、インバウンドともに）

## 5. 初心者がまず覚えるべきポイント

- IGWはVPCごとに1つ作成可能（複数作成不可）
- IGWはリージョン全体で利用可能（AZに依存しない）
- パブリックIPを持つリソースのみがIGW経由でインターネットにアクセス可能
- プライベートサブネットからインターネットにアクセスする場合はNAT Gatewayを使用
- IGWを削除する前に、関連するルートテーブルからルートを削除する必要がある

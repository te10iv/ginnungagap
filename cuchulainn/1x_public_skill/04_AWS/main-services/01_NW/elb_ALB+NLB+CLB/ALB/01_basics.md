# Application Load Balancer (ALB)：まずこれだけ（Lv1）

## このサービスの一言説明
- Application Load Balancer (ALB) は「**HTTP/HTTPSトラフィックを複数のターゲットに振り分けるロードバランサー**」を提供するAWSサービス

## ゴール（ここまでできたら合格）
- ALBを **作成できる**
- **ターゲットグループとリスナーの役割を説明できる**
- 「この構成にはALBが必要」と判断できる

## まず覚えること（最小セット）
- **ALB（Application Load Balancer）**：L7ロードバランサー（HTTP/HTTPS）
- **ターゲットグループ**：振り分け先のEC2/ECS/Lambda等をグループ化
- **リスナー**：どのポート（80/443）で受け付けるか
- **ヘルスチェック**：ターゲットの正常性を監視（/healthパス等）
- **パスベースルーティング**：URLパス（/api, /admin）で振り分け先を変更

## できるようになること
- □ マネジメントコンソールでALBを作成できる
- □ ターゲットグループにEC2を登録できる
- □ リスナールール（HTTP→HTTPS リダイレクト）を設定できる
- □ ヘルスチェックの設定ができる

## まずやること（Hands-on）
- 2つのEC2インスタンス（異なるAZ）を起動
- ターゲットグループを作成し、EC2を登録
- ALBを作成（Internet-facing、2つのPublicサブネット）
- リスナー（HTTP:80）を設定してアクセス確認

## 関連するAWSサービス（名前だけ）
- **EC2**：ALBのターゲット
- **ECS / Fargate**：ALBのターゲット
- **Lambda**：ALBのターゲット（HTTP API）
- **Route 53**：ALBにエイリアスレコード設定
- **ACM**：HTTPS化の証明書
- **WAF**：セキュリティフィルタリング

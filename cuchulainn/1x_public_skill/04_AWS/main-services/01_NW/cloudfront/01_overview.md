# CloudFront 概要

## 1. サービスの役割

Amazon CloudFrontは、コンテンツ配信ネットワーク（CDN）サービスです。静的コンテンツや動的コンテンツを世界中のエッジロケーションにキャッシュし、ユーザーに最も近い場所から高速に配信します。

## 2. 主なユースケース

- 静的ウェブサイトの高速配信（HTML、CSS、JavaScript、画像など）
- 動画コンテンツの配信
- APIのレスポンス時間の改善
- グローバルなアプリケーションのパフォーマンス向上
- DDoS攻撃の緩和（AWS Shieldと連携）
- カスタムドメインとSSL/TLS証明書の設定

## 3. 他サービスとの関係

- **S3**: オリジンとして静的コンテンツを配信
- **ELB, EC2**: オリジンとして動的コンテンツを配信
- **API Gateway, Lambda**: オリジンとしてAPIを配信
- **Route 53**: カスタムドメインのDNS設定
- **ACM**: SSL/TLS証明書の管理
- **WAF**: Webアプリケーションファイアウォールとの連携

## 4. 料金のざっくりイメージ

- データ転送: 最初の10TBまで$0.085/GB（リージョンにより異なる）
- HTTP/HTTPSリクエスト: $0.0075/10,000リクエスト
- 無料利用枠: 最初の50GBのデータ転送、2,000,000のHTTP/HTTPSリクエスト
- **注意**: オリジンへのデータ転送料金も別途発生

## 5. 初心者がまず覚えるべきポイント

- 世界中のエッジロケーションからコンテンツを配信
- キャッシュTTLを設定してオリジンへのリクエストを削減
- オリジンとしてS3、ELB、EC2、カスタムオリジンが使用可能
- CloudFront FunctionsやLambda@Edgeでエッジでの処理が可能
- カスタムドメインとSSL/TLS証明書（ACM）の設定が可能
- アクセスログやリアルタイムログでトラフィックを監視可能

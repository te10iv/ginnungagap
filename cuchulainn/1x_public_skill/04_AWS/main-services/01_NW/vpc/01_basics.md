# Amazon VPC：まずこれだけ（Lv1）

## このサービスの一言説明
- Amazon VPC は「**AWS上に独自のプライベートネットワーク空間を作る**」ための AWS サービス

## ゴール（ここまでできたら合格）
- VPCを **作れる**
- **使っている構成を見て何をしているか分かる**
- 「このシステムにはVPCが必要」と判断できる

## まず覚えること（最小セット）
- **VPC**：AWS上の仮想ネットワーク空間（例：10.0.0.0/16）
- **CIDR ブロック**：VPCのIPアドレス範囲（プライベートIPを使う）
- **サブネット**：VPC内をさらに分割したネットワーク（Public/Private）
- **Internet Gateway (IGW)**：VPCとインターネットをつなぐゲート
- **ルートテーブル**：通信の道順を決めるルール集

## できるようになること
- □ マネジメントコンソールでVPCを作成できる
- □ CIDRブロックの意味を説明できる
- □ サブネットの作成と配置ができる
- □ インターネット接続の有無を設定できる

## まずやること（Hands-on）
- マネコンでVPCを1つ作る（例：10.0.0.0/16）
- Publicサブネットを1つ作る
- Internet Gatewayをアタッチする
- ルートテーブルで0.0.0.0/0 → IGWを設定
- EC2を起動して疎通確認

## 関連するAWSサービス（名前だけ）
- **Subnet**：VPCを分割するネットワーク
- **Route Table**：通信経路を制御
- **Internet Gateway**：インターネット接続
- **NAT Gateway**：Private SubnetからのInternet接続
- **Security Group**：インスタンスレベルのファイアウォール
- **Network ACL**：サブネットレベルのファイアウォール

# VPC 概要

## 1. サービスの役割

VPC（Virtual Private Cloud）は、AWSクラウド内に論理的に分離された仮想ネットワーク環境を提供するサービスです。独自のIPアドレス範囲、サブネット、ルーティングテーブル、ゲートウェイを定義でき、オンプレミス環境と同様のネットワーク制御が可能です。

## 2. 主なユースケース

- プライベートなクラウド環境の構築
- マルチティアアーキテクチャの実現（Web層、アプリ層、DB層の分離）
- オンプレミス環境とのハイブリッド接続
- セキュリティ境界の設定とネットワークアクセス制御
- 複数のVPC間の接続とネットワーク分離

## 3. 他サービスとの関係

- **Subnet**: VPC内にサブネットを作成してリソースを配置
- **Route Table**: サブネットのトラフィックルーティングを制御
- **Internet Gateway**: パブリックサブネットからインターネットへの接続
- **NAT Gateway**: プライベートサブネットからインターネットへのアウトバウンド接続
- **VPC Endpoint**: AWSサービスへのプライベート接続
- **Security Group / NACL**: ネットワークレベルのセキュリティ制御
- **EC2, RDS, Lambda**: VPC内で動作する各種AWSサービス

## 4. 料金のざっくりイメージ

- VPC自体は無料
- 関連するリソース（NAT Gateway、VPC Endpoint、Transit Gatewayなど）に料金が発生
- データ転送料金が発生する場合あり

## 5. 初心者がまず覚えるべきポイント

- VPCはリージョンごとに作成される（リージョン内の複数AZにまたがれる）
- CIDRブロックでIPアドレス範囲を定義（例: 10.0.0.0/16）
- デフォルトVPCが各リージョンに存在するが、カスタムVPCの作成を推奨
- パブリックサブネットとプライベートサブネットの違いを理解する
- セキュリティグループとNACLの違いを理解する（SGはインスタンスレベル、NACLはサブネットレベル）

# Transit Gateway 概要

## 1. サービスの役割

Transit Gatewayは、複数のVPCやオンプレミス環境を中央のハブとして接続するマネージドサービスです。VPC Peeringの複雑なメッシュ接続を簡素化し、スケーラブルで管理しやすいネットワーク構成を実現します。

## 2. 主なユースケース

- 複数のVPC間の接続（ハブ&スポーク構成）
- オンプレミス環境と複数VPCの接続
- リージョン間のVPC接続（Transit Gateway Peering）
- 大規模なマルチVPC環境のネットワーク管理
- セグメント化されたネットワークアーキテクチャの実現

## 3. 他サービスとの関係

- **VPC**: Transit GatewayにVPCをアタッチ
- **VPN**: Site-to-Site VPN経由でオンプレミス環境を接続
- **Direct Connect**: Direct Connect Gateway経由でオンプレミス環境を接続
- **Route Table**: Transit Gatewayのルートテーブルでルーティングを制御
- **EC2, RDS**: 各VPC内のリソースがTransit Gateway経由で通信

## 4. 料金のざっくりイメージ

- Transit Gateway: 約$0.05/時間（約$36/月）
- VPCアタッチメント: 約$0.05/時間（約$36/月） × アタッチ数
- データ転送料金: GBあたり$0.02（同一リージョン内）
- **注意**: VPCが多くなるとコストが高くなる

## 5. 初心者がまず覚えるべきポイント

- VPC Peeringの代替として、複数VPC接続を簡素化
- ハブ&スポーク構成で、VPC間の接続を一元管理
- ルートテーブルで細かいルーティング制御が可能
- リージョンごとに作成する必要がある
- リージョン間接続にはTransit Gateway Peeringを使用
- セグメント化（ルートテーブル分離）でセキュリティを向上
- コストが高いので、本当に必要な場合のみ使用

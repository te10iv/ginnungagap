# Route 53 概要

## 1. サービスの役割

Amazon Route 53は、高可用性でスケーラブルなDNS（Domain Name System）サービスです。ドメイン名をIPアドレスに変換するだけでなく、ヘルスチェック、トラフィックルーティング、ドメイン登録などの機能を提供します。

## 2. 主なユースケース

- ドメイン名のDNS解決（Aレコード、CNAMEレコードなど）
- ヘルスチェックに基づく自動フェイルオーバー
- 地理的位置に基づくルーティング（Geolocation Routing）
- レイテンシーに基づくルーティング（Latency-based Routing）
- 重み付けルーティング（Weighted Routing）
- ドメインの登録と管理

## 3. 他サービスとの関係

- **EC2, ELB**: Route 53でDNSレコードを設定してアクセス
- **CloudFront**: CloudFrontディストリビューションのエイリアスレコード
- **S3**: 静的ウェブサイトホスティングのドメイン設定
- **API Gateway**: カスタムドメインの設定
- **ACM**: SSL/TLS証明書の管理と連携

## 4. 料金のざっくりイメージ

- ホストゾーン: 約$0.50/月
- クエリ: 最初の10億クエリまで$0.40/100万クエリ
- ヘルスチェック: 約$0.50/月（基本）+ エンドポイントごとに追加料金
- ドメイン登録: ドメインごとに年額料金（例: .comで約$12/年）

## 5. 初心者がまず覚えるべきポイント

- AWSの主要なDNSサービス
- エイリアスレコードでAWSリソース（ELB、CloudFrontなど）を直接指定可能
- ヘルスチェック機能で自動フェイルオーバーが可能
- 複数のルーティングポリシー（Simple、Weighted、Latency、Geolocation、Failover、Multivalue）
- プライベートホストゾーンでVPC内のDNS解決も可能
- ドメイン登録も可能（ただし、他のレジストラの方が安い場合もある）

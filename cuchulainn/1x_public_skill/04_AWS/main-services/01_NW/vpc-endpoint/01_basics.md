# VPC Endpoint：まずこれだけ（Lv1）

## このサービスの一言説明
- VPC Endpoint は「**AWSサービスにインターネットを経由せずプライベート接続**」を提供するAWSサービス

## ゴール（ここまでできたら合格）
- VPC Endpointを **作成できる**
- **ゲートウェイ型とインターフェース型の違いを説明できる**
- 「このAWSサービスへのアクセスにはVPCエンドポイントが有効」と判断できる

## まず覚えること（最小セット）
- **VPC Endpoint**：AWSサービスへのプライベート接続（インターネット不要）
- **ゲートウェイ型**：S3・DynamoDB専用（無料、ルートテーブルで設定）
- **インターフェース型（PrivateLink）**：その他AWSサービス（有料、ENI作成）
- **セキュリティグループ**：インターフェース型で通信制御
- **エンドポイントポリシー**：アクセス制御（IAMポリシー）

## できるようになること
- □ マネジメントコンソールでVPCエンドポイントを作成できる
- □ ゲートウェイ型とインターフェース型を選択できる
- □ ルートテーブルまたはサブネットに関連付けできる
- □ S3やECRにプライベート接続できる

## まずやること（Hands-on）
- マネコンでS3用のゲートウェイ型エンドポイントを作成
- ルートテーブルに関連付け
- PrivateサブネットのEC2からS3アクセス確認（aws s3 ls）
- VPC Flow Logsで通信がNAT Gateway経由でないことを確認

## 関連するAWSサービス（名前だけ）
- **S3 / DynamoDB**：ゲートウェイ型エンドポイント対象
- **ECR / SSM / CloudWatch / Secrets Manager**：インターフェース型エンドポイント対象
- **Route Table**：ゲートウェイ型エンドポイントで設定
- **Security Group**：インターフェース型エンドポイントで通信制御
- **NAT Gateway**：VPCエンドポイント使用でNAT Gateway不要に

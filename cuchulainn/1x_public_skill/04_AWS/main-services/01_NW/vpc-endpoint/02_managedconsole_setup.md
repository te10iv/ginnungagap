# VPC Endpoint マネージドコンソールでのセットアップ

## 作成するもの

VPC Endpoint（Gateway型）を作成し、プライベートサブネットからS3にプライベート接続できるようにします。インターネット経由せずにS3にアクセスできるため、セキュリティが向上し、NAT Gatewayのコストを削減できます。

## セットアップ手順

1. **VPC Endpointコンソールを開く**
   - VPCコンソールで「エンドポイント」を選択

2. **VPC Endpointを作成**
   - 「エンドポイントを作成」をクリック
   - **名前タグ**: `s3-endpoint` を入力
   - **サービスカテゴリ**: AWSサービスを選択
   - **サービス**: `com.amazonaws.region.s3` を検索して選択
   - **VPC**: 作成済みのVPCを選択
   - **ルートテーブル**: プライベートサブネットのルートテーブルを選択
   - **ポリシー**: フルアクセス（デフォルト）を選択
   - 「エンドポイントを作成」をクリック

3. **確認**
   - エンドポイントの状態が「利用可能」になっていることを確認
   - ルートテーブルに自動的にルートが追加されていることを確認

## 補足

- Gateway型はS3とDynamoDBのみ対応（無料）
- Interface型はより多くのAWSサービスに対応（有料）
- ポリシーでアクセス制御が可能です

# 概要

## 1. 要点まとめ

- Pythonは読みやすく書きやすい高水準プログラミング言語
- インタプリタ型言語で、コンパイル不要で実行できる
- 動的型付け言語（変数の型を明示的に宣言しない）
- インデント（字下げ）が構文の一部（ブロックの区切りに使用）
- 豊富な標準ライブラリとサードパーティライブラリが利用可能
- オブジェクト指向、関数型、手続き型など複数のプログラミングパラダイムに対応

## 2. 詳細解説

Pythonは1991年にGuido van Rossumによって開発されたプログラミング言語です。コードの可読性を重視して設計されており、「Pythonic」という言葉があるほど、シンプルで美しいコードを書くことができます。

Pythonはインタプリタ型言語のため、コンパイルという手順を経ずに、ソースコードを直接実行できます。これにより開発サイクルが短縮され、学習者にとっては理解しやすい環境を提供します。

また、Pythonは動的型付け言語です。変数を宣言する際に型を指定する必要がなく、実行時に自動的に型が決定されます。これにより柔軟なプログラミングが可能ですが、型エラーに注意が必要です。

Pythonの特徴的な構文として、インデント（字下げ）がブロック構造を表すことが挙げられます。他の多くの言語では`{}`や`begin/end`を使いますが、Pythonではインデントでコードブロックを表現します。これにより、コードが自然と読みやすくなります。

## 3. サンプルコード

```python
# 最初のPythonプログラム
print("Hello, World!")

# 変数の使用（型宣言不要）
name = "Python"
version = 3.11
print(f"{name} version {version}")

# インデントによるブロック構造
if version >= 3.0:
    print("Python 3系です")
    print("このブロックはインデントで区切られています")
```

## 4. 注意点・落とし穴

- **インデントエラー**: インデントが統一されていないと`IndentationError`が発生します。スペース4つまたはタブ1つで統一する必要があります（PEP 8ではスペース4つを推奨）
- **型の混同**: 動的型付けのため、意図しない型の変数が混入しやすいです。型チェックツール（mypy等）の使用を検討しましょう
- **バージョン違い**: Python 2とPython 3は互換性がありません。現在はPython 3系を使用するのが標準です
- **グローバル変数の扱い**: 関数内でグローバル変数を変更する場合は`global`キーワードが必要です

## 5. 演習問題

1. 自分の名前を表示するプログラムを作成してください
2. 変数`age`に年齢を代入し、「私は○歳です」と表示するプログラムを作成してください
3. インデントを使って、条件分岐を含むプログラムを作成してください

## 6. 解答例

```python
# 1. 名前を表示
name = "山田太郎"
print(f"私の名前は{name}です")

# 2. 年齢を表示
age = 25
print(f"私は{age}歳です")

# 3. 条件分岐（インデント使用）
score = 85
if score >= 80:
    print("優秀です")
    print("おめでとうございます")
else:
    print("もう少し頑張りましょう")
```

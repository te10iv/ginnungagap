# [技術分野] 学習に役立つ参考資料

cursolに、この技術分野に役立つ参考資料を列挙してもらいたい。

▼依頼内容
あなたはありとあらゆるIT分野のプロです
かつ、爆速および最短でIT微経験者を中級者までに育てられるプロの講師です。

以下の条件をすべて満たす形で、
この技術分野学習に役立つ参考資料を、可能な限り網羅的に列挙してください。
 ＊情報は今日現在の時点で最新のものを優先

各技術分野ディレクトリ（NW、Linux、AWS、PG、Dockerなど）の直下にstudy-materialsというディレクトリを作り、その中にreferences.mdというマークダウンファイル一つに情報を記載してください。


# 目的

- [技術分野] に **最短で中級者となるための学習リソース全集**を作成する
- 初学者〜中級者が「何を選べばよいか迷わない」状態を作る
- エンジニア視点で **実務に役立つかどうか**も評価する
  - 時間は有限であり無限ではないので、「まずはこれだけ知ればOK」「最低限はこれ」「大体の案件で使われる技術」「遭遇頻度の低いレアケース（よほどの大規模環境じゃないと使わない）」といった点を意識すること


# 出力ルール（重要）

- 出力は **Markdown形式**
- 各カテゴリごとに **複数件（最低3〜5件以上、可能ならそれ以上）**
- 情報は可能な限り **最新・定番・評価が高いもの**を優先
- 不明な項目は「不明」と明記（推測で書かない）
- 日本語教材・英語教材は分けてよい
- URLは公式・販売元・目次が確認できるページを優先

## URL に関する厳守ルール（最重要）

- URL は **実在し、現在も正常に表示できるページのみ**記載すること
- **存在確認ができない URL は絶対に記載しない**
- URL が不明な場合は、無理に推測せず「不明」と記載すること

## URL不明時の対応
- 正確性を最優先する
- URL が 1 件も記載されなくても問題ない
- 無理に埋めず「不明」と記載する


# 評価観点（全リソース共通）

各教材には必ず以下を含めること：

- 概要（どのレベル向けか／この技術分野の中のどれに強いか）
- 評価
  - 良い点（具体的に）
  - 悪い点（正直に）
- 「この教材が特に向いている人」

### Udemy 動画の URL ルール（厳守）

- URL は **Udemy のコーストップページのみ**記載すること
- 以下は禁止
  - 講義単体（/lecture/xxxxx）

- なぜこのルールが必要か
  - /lecture/ は高確率で無効化
  - セール URL は期限切れ
  - コーストップは 最も長生き

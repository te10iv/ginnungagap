# どこで何を確認するか（git確認コマンドのタイミング）

▼▼▼最低限

1)
git clone 直後：
　git status（ブランチ & clean 確認）

2)
ブランチ作成後：
　git status（今いるブランチ確認）

3)
編集したあと：
　git status
　git diff

4)
git add したあと：
　git status
　git diff --staged

5)
git commit したあと：
　git status
　git push 前：

6)
（できれば）git pull --rebase origin main

7)
その後 git push origin main


▼▼▼完全版
1)
git clone 
　cd リポジトリ
  git status        # ★ 今どのブランチ？ クリーン？
  git remote -v     # （余裕あれば）どのリポジトリから取ってきたか確認

2)
git switch -c feature/something
  git status   # ブランチ名が feature/something になってるか確認 (今どのブランチにいるかの確認はめちゃ大事!!)
　　　　　　　　　# main のまま編集 → そのまま push → あちゃー、がよくある…。　　　

3)
ソースコード編集
 git status # 何が変わったか（untracked / modified）
 git diff  # どこを変えたか（working vs indexの比較）

    ## ここは「1回だけ」じゃなくて、ちょっとまとまった編集をしたら癖で打つくらいでOK!!

4)
git add
 git status
 git diff --staged  # 次の commit に乗る内容の最終確認

   ## ここで「いらないファイルまで add しちゃった!!」「README まだ途中なのに add してた!!」に気づける。

5)
git commit前の確認★★★
　git status          # staged / not staged の最終確認
  git diff --staged   # commit 内容の最終チェック

6)
git commit -m "~~~" 
  git status          # clean になったか確認
  git log --oneline -n 3   # 直近の履歴を見る


7)
push 前に、できれば一回 pull. !!!特にチーム開発だとここ重要。!!!
  git pull --rebase origin main   # or origin ブランチ名
  git push origin main            # or origin feature/xxx
    ## 自分が作業している間にリモートが進んでいることがあるので、その場合 pull なしで push しようとして怒られることが多い。

8)
git push origin main
